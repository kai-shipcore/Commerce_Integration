# 📦 SKU Planning 사용 설명서

> **대상**: 재고·발주 담당자  
> **URL**: `/planning/sku-forecasts`  
> **목적**: SKU별 판매 추이, 재고 현황, 입고 예정 컨테이너, 발주 권고 수량을 한 화면에서 빠르게 확인

---

## 1. 화면 전체 구성

```
┌─────────────────────────────────────────────────────────────────────┐
│  SKU Planning               [마지막 동기화 시각]  [Containers 상태]  [Sync] │  ← 상단 헤더
├──────────────────────────┬──────────────────────────────────────────┤
│  [Seat Cover] [Car Cover] [Floor Mat]         │  SKU명 + 카테고리 배지 + 긴급도 │
│  ┌───────────────────┐   │  MOQ · Case · CBM · Next ETA        │  ← SKU 헤더
│  │ 🔍 Search SKU...  │   ├──────────────────────────────────────┤
│  └───────────────────┘   │  📊 현재재고  📈 일평균  📅 재고일수  │  ← KPI 카드
│  [All][●Crit][▲Watch]    │     📥 입고  🔮 예상재고             │
│  [High][Low]             ├──────────────────────────────────────┤
│  ┌───────────────────┐   │  [Sales Analysis] [Inventory&Inbound]│
│  │ Master SKU ▲  SOD │   │  [Container Recommendation]          │  ← 탭
│  ├───────────────────┤   ├──────────────────────────────────────┤
│  │ CA-XXX-BK   544d  │   │  (선택한 탭 콘텐츠)                  │
│  │ CA-YYY-GR    30d  │   │                                      │
│  │  ...              │   │                                      │
│  └───────────────────┘   │                                      │
└──────────────────────────┴──────────────────────────────────────────┘
   ← 왼쪽: SKU 목록 패널 →         ← 오른쪽: SKU 상세 패널 →
```

- **왼쪽 패널**: 제품 카테고리 탭 + 검색 + 긴급도 필터 + SKU 목록
- **오른쪽 패널**: 선택한 SKU의 헤더 정보 + KPI 카드 + 3개 탭(판매/재고/발주)

## 2. 상단 헤더

| 항목 | 설명 |
|------|------|
| **마지막 동기화 시각** | "Synced 2026-05-29 18:39" 형식으로 마지막 데이터 로드 시각 표시 |
| **컨테이너 상태** | "Loading containers…" / "Containers ready" / "Containers pending" |
| **Sync 버튼** | 클릭하면 모든 데이터를 새로 불러옵니다. 로딩 중에는 비활성화됩니다. |

> 💡 페이지에 처음 접속하면 **컨테이너 상세 데이터가 자동으로 로딩**됩니다. 잠시 기다리면 "Containers ready"로 바뀝니다.

---

## 3. 왼쪽 패널: SKU 목록

### 3-1. 제품 카테고리 탭

화면 상단에 세 가지 탭이 있습니다. 탭을 클릭하면 해당 카테고리의 SKU만 목록에 표시됩니다.

| 탭 | 카테고리 |
|----|---------|
| **Seat Cover** | 시트커버 |
| **Car Cover** | 차량커버 |
| **Floor Mat** | 플로어매트 |

각 탭 옆의 숫자는 해당 카테고리에 속한 **SKU 총 개수**입니다.  
탭을 바꾸면 검색어도 자동으로 초기화됩니다.

---

### 3-2. 검색창

- SKU 코드(예: `CA-CL-CH`) 또는 컨테이너 이름으로 검색합니다.
- 대소문자를 구분하지 않으며, 부분 일치도 됩니다.
- 텍스트를 입력하면 오른쪽에 **× 버튼**이 나타납니다. 클릭하면 검색어가 지워집니다.

---

### 3-3. 긴급도 필터 (필터 버튼)

검색창 아래에 6개의 필터 버튼이 있습니다.  
각 버튼 오른쪽의 숫자는 현재 카테고리 탭에서 해당 조건에 맞는 SKU 수입니다.

| 버튼 | 색상 | 조건 |
|------|------|------|
| **전체** | 파랑 (활성 시) | 모든 SKU 표시 (기본값) |
| **긴급** | 🔴 빨강 | 품절 예상일(SOD)이 30일 이내인 SKU |
| **주의** | 🟡 앰버 | 품절 예상일(SOD)이 31~60일인 SKU |
| **정상** | 🟢 초록 | 품절 예상일(SOD)이 60일 초과인 SKU (재고 여유) |
| **백오더** | 회색 | 백오더(재고 마이너스) 상태인 SKU |
| **발주 필요** | 보라 | 권장 발주 수량(Rec. Qty)이 0보다 큰 SKU |

> 💡 Critical 먼저 확인해서 긴급 발주가 필요한 SKU를 먼저 처리하세요!

---

### 3-4. SKU 목록

SKU 목록은 4개 컬럼으로 구성됩니다.

| 컬럼 | 설명 |
|------|------|
| **Master SKU** | SKU 코드 |
| **Stock** | 현재 전체 재고량 |
| **Avg** | 일평균 판매량 (소수점 2자리) |
| **SOD** | 재고 소진 예상일(일수). 긴급도에 따라 색상 배지로 표시 |

**SOD 배지 색상**:
- 🔴 빨간색: 30일 이내 (Critical)
- 🟡 앰버색: 31~60일 (Watch)
- 🟢 초록색: 60일 초과 (Healthy)

**정렬 기능**:  
컬럼 헤더를 클릭하면 해당 컬럼 기준으로 오름차순/내림차순 정렬합니다.  
활성 정렬 컬럼은 파란색으로 강조되며 ▲(오름차순) / ▼(내림차순) 아이콘이 표시됩니다.

**SKU 선택**:  
행을 클릭하면 오른쪽 패널에 해당 SKU의 상세 정보가 표시됩니다.  
선택된 행은 파란색 배경으로 강조됩니다.

---

## 4. 오른쪽 패널: SKU 상세

왼쪽에서 SKU를 선택하면 오른쪽 패널이 3개 영역으로 구성됩니다.

```
[SKU 헤더] → SKU명, 카테고리, 상태, 긴급도, 기본 스펙
[KPI 카드] → 현재재고, 일평균, 재고일수, 입고, 예상재고
[탭 영역]  → Sales Analysis / Inventory & Inbound / Inbound History
             / Purchase Recommendation / Demand Forecast
```

### 4-1. SKU 헤더

선택한 SKU의 기본 정보를 보여줍니다.

**좌측 - SKU 기본 정보:**
- **SKU 코드**: 큰 굵은 글씨로 표시
- **상품명**: 등록된 경우 SKU 코드 아래에 회색으로 표시

**우측 - 상태 배지 (3개):**

| 배지 | 내용 | 예시 |
|------|------|------|
| **카테고리** | 파란 배지. 해당 제품군 | `Seat Cover` |
| **상품 상태** | 회색 배지. 상품 분류 | `Original` / `Custom` / `Hold` |
| **긴급도** | 색상 배지 | 🔴 `Critical` / 🟡 `Watch` / 🟢 `Healthy` |

**하단 - 제품 스펙 (4개):**

| 항목 | 의미 |
|------|------|
| **MOQ** | 최소 주문 수량 |
| **Case** | 케이스당 수량 |
| **CBM** | 개당 부피 (㎥, 소수점 4자리) |
| **Next ETA** | 다음 입고 예정일 (없으면 `-`) |

---

### 4-2. KPI 카드 (5개)

SKU의 핵심 지표를 한눈에 볼 수 있는 카드입니다.

| 카드 | 계산식 | 설명 |
|------|--------|------|
| **Current Stock** | 현재 재고량 | "West {서부재고} / East {동부재고}" 표시 |
| **Daily Average** | 일평균 판매량 | "30D {30일 판매} units" 표시 |
| **Inv. Life** | SOD까지 남은 일수 | "Based on projected SOD" 표시. 데이터 없으면 `-` |
| **Inbound** | 선적 완료·패킹리스트 수령 컨테이너의 총 입고 예정 수량 | "Next ETA {날짜}" 표시 |
| **Projected** | 현재재고 + 입고 예정 + min(백오더, 0) | "CBM/unit {단위 부피}" 표시 |

> 💡 **Inv. Life**는 현재 재고와 확정 입고 수량을 합쳐서 현재 판매 속도로 나눈 **예상 재고 소진까지의 남은 일수**입니다.  
> 이 숫자가 작을수록 빨리 재고가 소진됩니다. 30일 이하면 즉시 발주 검토가 필요합니다.

## 5. 탭 1 — Sales Analysis (판매 분석)

### 5-1. Sales Trend (판매 추이 차트)

기간별 판매량을 막대 차트로 시각화합니다.

- 막대가 높을수록 판매량이 많습니다.
- 막대 위에 해당 기간의 판매 수량이 표시됩니다.
- 판매량이 0인 기간도 최소 높이(4%)로 표시됩니다.

**표시 기간**: 7일 / 15일 / 30일 / 60일 / 90일

---

### 5-2. West Sales / East Sales (서부·동부 판매 현황)

서부(West FBM)와 동부(East FBM) 각각의 기간별 누적 판매 수량을 표로 보여줍니다.

| 기간 | 설명 |
|------|------|
| 7D | 최근 7일 |
| 15D | 최근 15일 |
| 30D | 최근 30일 |
| 60D | 최근 60일 |
| 90D | 최근 90일 |

---

### 5-3. 평균 판매량 카드 (3개)

| 카드 | 의미 |
|------|------|
| **Previous Avg.** | 이전 기준 일평균 판매량 |
| **Real Avg.** | 실제 측정된 일평균 판매량 |
| **Current Avg.** | 현재 적용 중인 일평균 판매량 (파란색 강조) |

단위: `/d` (하루당)  
**Current Avg.** 카드는 파란 테두리와 파란 배경으로 강조 표시되어 현재 사용 중인 속도를 한눈에 알 수 있습니다.

---

## 6. 탭 2 — Inventory & Inbound (재고·입고 현황)

### 6-1. 재고 요약 카드 (4개)

| 항목 | 설명 |
|------|------|
| **West Stock** | 서부 창고 현재 재고량 |
| **East Stock** | 동부 창고 현재 재고량 |
| **Backorder** | 백오더 수량. **음수이면 빨간색**으로 표시 |
| **SOD** | 품절 예상일 + 남은 일수 |

---

### 6-2. 재고 적정성 게이지 바

재고 보유 수준을 시각적으로 나타내는 파란색 가로 막대입니다.

**계산 방식**: (예상 재고 ÷ 일평균 × 120일) × 100%

막대가 길수록 재고가 충분합니다. 막대가 짧으면 빠른 시일 내에 재고 보충이 필요합니다.

---

### 6-3. Inbound Containers (입고 예정 컨테이너)

**Shipped(선적 완료)** 또는 **Packing Received(패킹리스트 수령)** 상태의 컨테이너 중 이 SKU 입고 예정 수량이 있는 것들만 표시됩니다.  
(Draft 상태는 아직 확정되지 않은 발주 계획이므로 제외됩니다.)

각 컨테이너 행에는 아래 정보가 표시됩니다:

| 항목 | 설명 |
|------|------|
| **컨테이너명** | 컨테이너 이름. 아래에 현재 상태가 표시됩니다. |
| **ETA** | 예상 도착일 |
| **Qty** | 이 SKU의 입고 예정 수량 |
| **CBM** | 이 SKU가 컨테이너에서 차지하는 부피(㎥) |

> 💡 **컨테이너 행을 클릭하면** 해당 컨테이너의 상세 관리 페이지(`/planning/container-planning`)로 이동합니다!

**빈 상태 메시지**:
- 로딩 중: "Loading container details..."
- 활성 컨테이너 없음: "No active inbound containers."

---

### 6-4. 재고 합산 카드 (오른쪽, 4개)

| 카드 | 계산식 |
|------|--------|
| **Current Stock** | 현재 전체 재고량 |
| **Inbound** | 입고 예정 수량 (+접두사로 표시) |
| **Projected Stock** | 현재재고 + 입고 예정 + min(백오더, 0) |
| **Remaining / Mistake** | 잔여 수량 / 오류 수량 |

## 7. 탭 3 — Container Recommendation (발주 권고)

현재 재고 상황과 목표 재고 일수(90일)를 기반으로 **추가로 발주해야 할 수량을 자동 계산**해 줍니다.

### 7-1. 발주 권고 카드 (왼쪽)

| 항목 | 설명 |
|------|------|
| **Recommended Container Qty** | 권고 발주 수량 (큰 굵은 글씨, 4xl) |
| **Multiple** | 주문 단위 (예: "Multiple 5 units" = 5의 배수로만 주문 가능) |
| **Estimated CBM** | 권고 수량 × 개당 부피(m³) |

> ⚠️ 권고 수량이 0보다 크면 카드가 **파란 테두리와 파란 배경**으로 강조됩니다.  
> 0으로 표시되면 현재 재고와 입고 예정으로 90일치가 충분하다는 의미입니다.

---

### 7-2. 계산 과정 (오른쪽)

어떻게 권고 수량이 계산되었는지 8단계로 투명하게 보여줍니다.

| 항목 | 계산식 | 설명 |
|------|--------|------|
| **Target Days** | 90일 (고정) | 목표 재고 보유 기간 |
| **Daily Average** | 현재 일평균 | 현재 적용 중인 일평균 판매 속도 |
| **Target Stock** | 올림(일평균 × 90) | 90일치 판매를 위한 목표 재고량 |
| **Current Stock** | 현재 재고량 | 지금 창고에 있는 재고 |
| **Inbound** | 선적 완료 컨테이너 입고 예정 합계 | 들어올 예정인 수량 |
| **Backorder Impact** | min(백오더, 0) | 음수 재고분 반영 (0 이상이면 0으로 처리) |
| **Projected Stock** | 현재 + 입고 + 백오더 | 예상 가용 재고 |
| **Raw Need** | max(목표 - 예상, 0) | 부족분 (발주 필요량) (음수는 0으로 처리) |

**최종 계산**:
```
권고 수량 = Raw Need가 0이면 → 0
           Raw Need가 있으면 → 올림(Raw Need ÷ 주문 단위) × 주문 단위
```

**예시**:  
- 일평균 50개/일, 현재 재고 3,000개, 입고 예정 1,000개  
- 목표 재고 = 50 × 90 = 4,500개  
- 예상 재고 = 3,000 + 1,000 = 4,000개  
- Raw Need = 4,500 - 4,000 = 500개  
- 주문 단위 5이면 → 권고 수량 = 500개 (이미 배수)

---

## 8. 탭 5 — Demand Forecast (수요 예측)

### 8-1. 개요

수요 예측 탭은 통계 머신러닝 모델을 사용하여 이 SKU의 **향후 주간 판매량을 예측**합니다. 예측을 바탕으로 재고가 언제 소진될지 미리 파악하고 발주 타이밍을 계획할 수 있습니다.

탭에는 두 가지 모드가 있습니다:

| 모드 | 설명 |
|------|------|
| **Forward (예측 모드)** | 지금부터 미래 판매량을 예측합니다 |
| **Backtest (검증 모드)** | 과거 데이터로 모델이 얼마나 정확한지 확인합니다 |

---

### 8-2. Forward 모드 — 미래 예측

#### 예측 차트

차트에는 다음 요소가 표시됩니다:

| 요소 | 색상 | 설명 |
|------|------|------|
| **Actual demand (실제 수요)** | 파란색 실선 | 과거 실제 주간 판매 이력 |
| **Forecast (예측)** | 주황색 실선 | 모델이 예측한 향후 주간 판매량 |
| **P70 interval (P70 구간)** | 주황색 반투명 영역 | 예측의 불확실성 범위 (아래 설명 참고) |
| **Stockout (재고 소진 예상선)** | 분홍색 점선 | 현재 재고가 예측 판매량 기준으로 모두 소진되는 예상 날짜 |

> 💡 **P70 구간이란?** 예측은 하나의 숫자이지만 실제 판매는 그보다 높거나 낮을 수 있습니다. P70 구간은 "10번 중 약 7번은 실제 판매가 이 범위 안에 들어온다"는 의미입니다. 구간이 넓을수록 예측의 불확실성이 높습니다.

> 💡 **재고 소진선 (분홍색 점선)** 이 날짜 이전에 발주가 완료되고 입고가 이루어져야 품절을 막을 수 있습니다. 표시되지 않으면 예측 기간 내에 재고가 부족하지 않다는 뜻입니다.

차트는 **마우스 드래그로 좌우 이동(팬)** 하거나 **스크롤로 확대/축소** 할 수 있습니다. 이동 범위는 가장 오래된 판매 데이터 날짜부터 마지막 예측 날짜까지로 제한됩니다.

---

#### 컨트롤

**예측 주차 (Forecast weeks)**

예측할 기간을 주 단위로 설정합니다.

- 범위: 최소 1주 ~ 최대 52주 (약 1년)
- 기본값: 13주 (약 3개월)
- 숫자를 입력한 뒤 **Enter** 키를 누르거나 입력창 밖을 클릭하면 적용됩니다
- 저장된 예측 범위를 초과하면 서버가 예측을 새로 실행합니다 (5~30초 소요)

**History (이력 표시 기간)**

차트에서 과거 데이터를 얼마나 보여줄지 설정합니다.

- 달력으로 특정 날짜를 선택하거나, "4w", "12w" 등 주 단위로 입력할 수 있습니다
- **이력 기간을 줄여도 데이터가 삭제되지 않습니다** — 차트의 초기 뷰(줌)만 변경됩니다
- 차트에서 왼쪽으로 스크롤하면 더 오래된 이력을 볼 수 있습니다

**Model (예측 모델 선택)**

예측에 사용할 통계 모델을 선택합니다.

- 기본값 **Auto**: 이 SKU에 가장 적합한 모델이 자동으로 선택됩니다
- 다른 모델을 선택하면 서버가 예측을 새로 실행합니다 (5~30초 소요)
- 모델 종류는 아래 [8-4. 예측 모델 안내](#8-4-예측-모델-안내) 참고

|------|------|

### 8-3. Backtest 모드 — 과거 검증

#### 백테스트란?

백테스트는 **"과거 특정 시점에 예측했다면 실제와 얼마나 맞았을까"** 를 계산하는 방법입니다.

예를 들어 Cutoff(기준일)를 6개월 전으로 설정하고 Horizon(예측 기간)을 13주로 설정하면, 6개월 전 데이터만으로 13주 후까지 예측하고, 그 예측이 실제 판매와 얼마나 일치했는지 계산합니다. 이를 통해 모델이 이 SKU에 신뢰할 수 있는지 판단할 수 있습니다.

---

#### 컨트롤

| 항목 | 설명 |
|------|------|
| **Cutoff (기준일)** | 예측의 시작점. 이 날짜 이전 데이터를 학습에 사용하고, 이 날짜 이후 실제 판매와 예측을 비교합니다. 월요일 단위로 선택됩니다 |
| **Horizon (예측 기간)** | 기준일로부터 몇 주 앞을 예측할지 설정합니다 (1~52주) |
| **Train (학습 기간)** | 학습 데이터를 최근 N주로 제한합니다. 0은 전체 이력을 사용합니다. 이력이 너무 오래되어 현재 판매 패턴과 다를 경우 제한이 유용합니다 |
| **History (차트 표시)** | 기준일 이전 데이터를 차트에 얼마나 표시할지 설정합니다. Forward 모드와 마찬가지로 차트 줌만 변경되며 스크롤로 더 많은 이력을 볼 수 있습니다 |
| **Model 1** | 기본 예측 모델 선택 |
| **Model 2 (앙상블)** | 선택 사항. 두 번째 모델을 선택하면 두 모델 예측의 평균이 차트에 추가로 표시됩니다. 모델 간 앙상블로 예측을 보완할 수 있습니다 |
| **Run 버튼** | 위 설정으로 백테스트를 실행합니다. 최초 실행 시 5~30초 소요됩니다 |

> 💡 **Cutoff를 최근 날짜로** 설정할수록 현재 시장 상황에 가까운 검증이 됩니다. 오래된 날짜로 설정하면 더 긴 평가 기간을 볼 수 있습니다.

---

#### 백테스트 차트

| 요소 | 색상 | 설명 |
|------|------|------|
| **Actual demand** | 파란색 실선 | 실제 주간 판매 이력 전체 |
| **Forecast** | 주황색 실선 | 모델이 Cutoff 이후 예측한 값 |
| **P70 interval** | 주황색 반투명 영역 | 70% 확률 예측 구간 |

기준일(Cutoff) 이후 구간에서 **예측선과 실제 판매선이 얼마나 가까운지** 시각적으로 확인할 수 있습니다. 실제 판매가 P70 구간 안에 많이 들어올수록 모델 신뢰도가 높습니다.

---

#### 메타 정보 (백테스트 결과)

백테스트 실행 후 차트 아래에 정확도 지표들이 표시됩니다. 각 항목을 **클릭하면 상세 설명 팝업**이 나타납니다.

| 항목 | 설명 | 판단 기준 |
|------|------|----------|
| **MAE** | 평균 절대 오차 (Mean Absolute Error). 예측값과 실제값 차이의 평균. 단위는 수량(개) | 낮을수록 좋음 |
| **WAPE** | 가중 절대 백분율 오차 (Weighted Absolute Percentage Error). 오차를 퍼센트로 표현 | 10% 이하 우수 · 20% 이하 양호 · 그 이상 개선 필요 |
| **MASE** | 평균 절대 스케일 오차 (Mean Absolute Scaled Error). "지난 주 판매량을 그대로 예측값으로 쓰는 것"보다 얼마나 나은지 표현. 1.0 미만이면 모델이 기준보다 나음 | 1.0 미만이면 유효한 모델 |
| **P70 Coverage** | 실제 판매 중 P70 예측 구간 안에 들어온 비율. 70%에 가까울수록 구간 예측이 정확함 | 70% 근처가 이상적 |
| **Model** | 사용된 모델 이름 | — |
| **Train** | 학습에 사용된 주차 수 | — |
| **Eval** | 평가(검증)에 사용된 주차 수 | — |
| **Horizon actual** | 평가 기간 동안의 실제 총 판매량 | — |
| **Horizon forecast** | 평가 기간 동안의 예측 총 판매량 | — |
| **Difference** | 예측 총량 − 실제 총량. 양수면 과다 예측, 음수면 과소 예측 | 0에 가까울수록 좋음 |

---

### 8-4. 예측 모델 안내

특별한 이유가 없으면 **Auto**를 권장합니다. 아래 표는 각 모델의 특징을 간략히 설명합니다.

| 모델 | 특징 | 권장 상황 |
|------|------|----------|
| **Auto** | 교차 검증으로 이 SKU에 가장 정확한 모델을 자동 결정 | 일반 사용 (기본 권장) |
| **AutoARIMA** | 추세와 계절성을 자동으로 파악하는 통계 모델 | 연간 계절 패턴이 뚜렷한 제품 |
| **AutoETS** | 지수평활법 기반. 판매가 서서히 증가하거나 감소하는 추세에 강함 | 안정적이고 완만한 판매 흐름 |
| **AutoTheta** | 장기 추세 포착에 강한 모델 | 꾸준히 성장하거나 감소하는 제품 |
| **Window Avg** | 최근 N주의 평균을 예측값으로 사용 | 추세 없이 일정한 판매량 |
| **Historic Avg** | 전체 이력의 평균값을 사용 | 장기 평균이 안정적인 제품 |
| **Seasonal Naïve** | 작년 같은 주차 실적을 그대로 반복 | 강한 연간 계절성이 있는 제품 |
| **Naïve** | 지난 주 실적을 그대로 반복 | 매우 불규칙한 판매, 기준선 비교용 |

## 9. 긴급도 시스템 정리

이 페이지 전체에서 동일한 3단계 긴급도가 사용됩니다.

| 등급 | 조건 | 배지 색상 | 필터 버튼 |
|------|------|----------|----------|
| 🔴 **Critical** | SOD ≤ 30일 **또는** 백오더 상태 | 빨간색 | ● Critical |
| 🟡 **Watch** | SOD 31~60일 | 앰버색 | ▲ Watch |
| 🟢 **Healthy** | SOD > 60일 | 초록색 | High |
| ⬜ **Low** | 백오더 (back < 0) | 회색 | Low |

> 💡 **백오더**(Low/Critical)는 판매 주문이 재고보다 많은 상태입니다. 즉시 발주 처리가 필요합니다.

## 10. 자주 사용하는 시나리오

### 📌 시나리오 1: 긴급 발주 필요한 SKU 찾기

1. 제품 탭에서 카테고리 선택 (예: Seat Cover)
2. **● Critical** 필터 클릭 → 30일 이내 소진 예정 SKU만 표시
3. **SOD 컬럼 헤더 클릭** → 소진일 빠른 순으로 정렬
4. 가장 위의 SKU 클릭 → 오른쪽 패널에서 상세 확인
5. **Container Recommendation 탭** → 권고 발주 수량 확인

---

### 📌 시나리오 2: SKU 재고 및 입고 현황 파악

1. 목록에서 확인하고 싶은 SKU 클릭
2. 상단 KPI 카드로 **현재 재고**, **일평균 판매**, **재고 소진까지 남은 일수** 한눈에 확인
3. **Inventory & Inbound 탭** 클릭
4. 어떤 컨테이너에 몇 개가 실려 있는지, ETA가 언제인지 확인
5. 컨테이너 행 클릭 → 해당 컨테이너 상세 관리 페이지로 이동 가능

---

### 📌 시나리오 3: 재고가 낮지 않은데 발주 수량 확인

1. **High** 필터 클릭 → 재고 여유가 있는 SKU 확인
2. SKU 선택 → **Container Recommendation 탭** 확인
3. 권고 수량이 0이면 현재 상태로 90일치 재고가 충분
4. 권고 수량이 있으면 Inbound 물량 포함해도 조금 부족한 상황

---

### 📌 시나리오 4: 특정 SKU의 판매 추이 분석

1. 목록에서 SKU 클릭
2. **Sales Analysis 탭** 클릭
3. Sales Trend 차트로 기간별 판매 흐름 확인
4. West Sales / East Sales 표에서 채널별 판매량 비교
5. Current Avg.(현재 적용 일평균)와 Real Avg.(실제 측정값) 비교

---

### 📌 시나리오 5: 백오더 상태인 SKU 처리

1. **Low** 필터 클릭 → 백오더 SKU 표시
2. SKU 클릭 → KPI 카드에서 **Inv. Life** 확인 (음수이거나 매우 짧을 수 있음)
3. **Container Recommendation 탭** → 권고 발주 수량 확인 및 즉시 발주 조치

---

### 📌 시나리오 6: 수요 예측으로 발주 시점 계획

1. SKU 클릭 → **Demand Forecast 탭** 클릭
2. 차트에서 **분홍색 점선(Stockout 예상선)** 위치 확인
3. Stockout 날짜보다 **컨테이너 리드타임(일반적으로 30~60일)** 앞서 발주 필요
4. 메타 정보의 **Forecasted Stockout** 날짜로 기준일 확인
5. 예측 기간이 부족하면 Forecast weeks를 늘려 더 먼 미래까지 확인 (최대 52주)

---

### 📌 시나리오 7: 모델 정확도 확인 및 비교

1. **Demand Forecast 탭** → **Backtest** 모드 선택
2. **Cutoff** 날짜를 3~6개월 전으로 설정
3. **Horizon**을 발주 계획에서 중요한 기간(예: 13주)으로 설정
4. **Run** 클릭 → 메타 정보에서 **WAPE** 확인
5. WAPE가 높으면 다른 Model을 선택하거나 **Model 2**를 추가해 앙상블 효과 비교

## 12. 자주 묻는 질문 (FAQ)

**Q. SKU를 클릭했는데 오른쪽이 바뀌지 않아요.**  
A. 데이터 로딩 중일 수 있습니다. 상단에 "Loading…"이 사라진 후 다시 클릭해 주세요.

**Q. Inbound Qty가 있는데 Inbound Containers 목록이 비어 있어요.**  
A. 입고 예정 컨테이너가 "Shipped" 또는 "Packing Received" 상태여야 목록에 표시됩니다. Draft(계획 단계) 상태의 컨테이너는 목록에서 제외됩니다.

**Q. Container Recommendation의 권고 수량이 0인데 발주 안 해도 되나요?**  
A. 현재 재고 + 입고 예정으로 90일치가 충분하다는 의미입니다. 다만 판매 속도가 급변하거나 컨테이너 지연 가능성이 있다면 추가 검토를 권장합니다.

**Q. Inv. Life가 `-`로 표시돼요.**  
A. 품절 예상일(SOD) 데이터가 없거나, 일평균 판매량이 0인 경우입니다. 판매 실적이 없는 신규 상품일 가능성이 있습니다.

**Q. 필터를 All로 설정했는데 일부 SKU가 안 보여요.**  
A. 판매 실적이 없는 SKU는 기본적으로 숨겨집니다. 전체를 보려면 Planning Dashboard(`/planning/dashboard`)의 **⊞ Columns → Show Zero-Sales SKUs**를 활성화하면 됩니다.

**Q. 컨테이너를 클릭해도 이동이 안 돼요.**  
A. 컨테이너 ID가 등록되지 않은 경우 클릭이 비활성화됩니다. 데이터가 완전히 로딩된 후 (Containers ready 상태) 다시 시도해 보세요.

**Q. Sync를 눌렀는데 데이터가 오래 걸려요.**  
A. 1,000개 이상의 SKU 데이터를 로딩하므로 네트워크 상황에 따라 수 초가 걸릴 수 있습니다. "Synced HH:MM" 표시가 바뀌면 완료된 것입니다.

**Q. Demand Forecast 탭에 차트가 나타나지 않아요.**  
A. 예측 서버가 실행 중인지 확인하세요. 서버가 꺼져 있으면 차트가 표시되지 않습니다. 관리자에게 문의하거나 서버 재시작 후 다시 시도하세요.

**Q. 예측 주차를 13주에서 26주로 늘렸더니 로딩이 오래 걸려요.**  
A. 저장된 예측 범위를 초과하면 서버가 예측을 새로 실행합니다. 처음 실행 시 30초 내외가 소요될 수 있으며 이후에는 빠르게 로딩됩니다.

**Q. Backtest WAPE가 30% 이상인데 예측을 신뢰해도 되나요?**  
A. WAPE가 높을수록 예측 정확도가 낮습니다. 다른 모델을 시도하거나 Train 기간을 조정해 보세요. 판매량 자체가 매우 불규칙한 SKU는 어떤 모델을 써도 정확도가 낮을 수 있으며, 이 경우 예측을 참고용으로만 활용하는 것을 권장합니다.

**Q. Stockout 예상선이 차트에 보이지 않아요.**  
A. 예측 기간 내에 재고가 소진되지 않으면 Stockout 선이 표시되지 않습니다. 재고가 충분하다는 의미입니다.
