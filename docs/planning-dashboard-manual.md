# 📋 Demand Planning Dashboard 사용 설명서

> **대상**: 재고·발주 담당자  
> **URL**: `/planning/dashboard`  
> **목적**: SKU별 재고 현황, 판매 추이, 컨테이너 입고 예정, 품절 예상일(SOD)을 한눈에 파악하고 관리

---

## 1. 화면 전체 구성

```
┌──────────────────────────────────────────────────────────────────┐
│  [Product▾] [Type▾] [Status▾] [🔍 검색창] [⊞ Columns▾] [통계] │  ← 상단 툴바
├──────────────────────────────────────────────────────────────────┤
│  # │ Container Info │ CBM │ Back │ Status │ SKU │ ... (컬럼들) │  ← 헤더
├──────────────────────────────────────────────────────────────────┤
│  1 │                │     │      │        │     │              │
│  2 │                │     │      │        │     │              │  ← 데이터 행
│  ...                                                            │
└──────────────────────────────────────────────────────────────────┘
```

- **상단 툴바**: 카테고리 선택, 필터, 검색, 컬럼 설정, 날짜 선택, 데이터 동기화
- **헤더**: 컬럼 이름. 클릭하면 해당 컬럼으로 정렬됩니다.
- **데이터 행**: SKU별 재고·판매·입고 정보. 일부 셀은 직접 수정 가능합니다.

---

## 2. 상단 툴바 상세 안내

### 2-1. 카테고리 선택 (Product)

| 옵션 | 의미 |
|------|------|
| Floor Mat  | 플로어매트 제품 |
| Car Cover  | 차량커버 제품 |
| Seat Cover | 시트커버 제품 |

카테고리를 변경하면 해당 카테고리의 SKU만 표시됩니다.  
처음 접속하면 **Floor Mat**가 기본값입니다.

---

### 2-2. 상품 유형 필터 (Type)

| 옵션 | 의미 |
|------|------|
| All Types | 모든 상품 (기본값) |
| Original | 정규 상품만 |
| Custom | 커스텀 상품만 |

> 💡 유형 필터를 바꾸면 긴급도 필터가 초기화됩니다.

---

### 2-3. 긴급도 필터 (Status)

재고 상황을 긴급도별로 빠르게 걸러볼 수 있습니다.

| 옵션 | 색상 | 조건 |
|------|------|------|
| — All Status | 기본 | 전체 표시 |
| Critical (긴급) | 🔴 빨간색 | 품절 예상일(SOD)이 30일 이내 |
| Warning (주의) | 🟡 주황색 | 품절 예상일(SOD)이 31~60일 이내 |
| BackOrder | 🔵 파란색 | 재고가 마이너스(백오더 상태) |

> ⚠️ **Critical(긴급) 상태 SKU는 데이터 행 배경이 연분홍색으로 표시**되어 눈에 띄게 구별됩니다.

---

### 2-4. 검색창

- SKU명 또는 컨테이너명으로 검색합니다.
- 대소문자를 구분하지 않으며, 부분 일치도 됩니다.
- 텍스트를 입력하면 오른쪽에 **× 버튼**이 나타납니다. 클릭하면 검색이 초기화됩니다.

**예시**: `CA-CL` 입력 시 해당 문자를 포함하는 SKU가 모두 표시됩니다.

---

### 2-5. ⊞ Columns 버튼 (컬럼 설정)

클릭하면 팝오버가 열려 화면에 표시할 컬럼을 세밀하게 조정할 수 있습니다.

#### 📌 Quick Preset (빠른 설정)

| 프리셋 | 표시되는 컬럼 그룹 |
|--------|-------------------|
| **All** | 컨테이너 컬럼을 제외한 모든 그룹 |
| **Core** | 기본정보 + 재고 + 30D 판매 + 전체 평균 + 입고·SOD |
| **Compact** | 기본정보 + 재고 + 전체 평균 + 입고·SOD (최소화) |

#### 📌 Column Groups (컬럼 그룹 선택)

체크박스로 보고 싶은 그룹만 켜고 끌 수 있습니다.

| 그룹명 | 포함 내용 |
|--------|----------|
| Inventory | 서부·동부·전체 재고 |
| 📈 West Sales | 서부 FBM 판매 (7일~90일) |
| 📈 East Sales | 동부 FBM 판매 (7일~90일) |
| 〜 W Avg | 서부 일평균 판매량 (이전/실제/현재) |
| 〜 E Avg | 동부 일평균 판매량 |
| FBA Avg | FBA 일평균 판매량 |
| 🗓 30D Sales | 30일 채널별 판매 합계 |
| 〜 Total Avg | 전체 채널 일평균 (이전/실제/현재) |
| 🚢 Inbound/SOD | 입고 수량, 컨테이너 목록, 다음 ETA, 품절예상일 |
| 📋 Container 컬럼 | 컨테이너별 상세 데이터 (로딩 필요) |

#### 📌 Options (추가 옵션)

| 옵션 | 설명 |
|------|------|
| Show Remaining | 컨테이너 컬럼에서 "잔여 수량" 표시 여부 |
| Show Mistake | 컨테이너 컬럼에서 "오류 수량" 표시 여부 |
| Show Zero-Sales SKUs | 90일간 판매 실적이 없는 SKU도 표시 여부 (기본: 숨김) |

#### 📌 📌 Freeze Column (컬럼 고정)

스크롤해도 항상 보이게 고정할 컬럼을 드롭다운에서 선택합니다.  
기본값은 **S.O.D(품절예상일)** 컬럼입니다.  
Reset 버튼을 누르면 기본값으로 돌아옵니다.

> 💡 고정된 컬럼 오른쪽 경계선이 **청록색(━)** 으로 표시되어 구분됩니다.

#### 📌 Reset Column Widths

컬럼 너비를 모두 기본 크기로 복원합니다.

---

### 2-6. 기준 날짜 (As of)

- 기본값은 **오늘 날짜**입니다.
- 날짜를 바꾸면 그 날 기준의 과거 데이터를 조회할 수 있습니다.
- 오늘이 아닌 날짜가 선택되면 **파란색**으로 강조되고 **Today** 버튼이 나타납니다.

> 💡 과거 특정 시점의 재고·판매 상황을 재현하고 싶을 때 사용합니다.

---

### 2-7. Sync (데이터 동기화)

- 클릭하면 최신 데이터를 서버에서 새로 불러옵니다.
- 로딩 중에는 버튼이 **회색(Loading…)** 으로 바뀝니다.
- 데이터 동기화 후 마지막 동기화 시각이 "Synced 2025-05-29 18:39" 형식으로 표시됩니다.

---

### 2-8. CSV 내보내기

- 현재 화면에 필터링된 SKU 목록을 엑셀 파일로 내보냅니다.
- 파일명: `planning_YYYY-MM-DD.csv`
- 포함 컬럼: SKU, 재고(서부/동부/전체), Back, 상태, W30D, E30D, Total30D, 일평균(현재), 입고량, 컨테이너 목록, 다음 ETA, SOD

---

### 2-9. Velocity Mode (Link / Custom)

| 모드 | 설명 |
|------|------|
| **Link** | 연동된 판매 데이터 기반 계산 |
| **Custom** | 생산 데이터 기반 계산 |

> 모드를 바꾸면 일평균 판매량과 SOD가 재계산됩니다.

## 3. 컬럼 상세 설명

### 3-1. 기본 정보 컬럼 (항상 표시)

| 컬럼 | 설명 |
|------|------|
| **#** | 행 번호 (1부터 시작) |
| **Container Info.** | 컨테이너·좌석·색상 등 상품 정보. 길면 클릭하면 전체 내용과 복사 버튼이 나타납니다. |
| **CBM** | 상품 1개당 부피(㎥). **파란 밑줄**이 있으면 클릭해서 수정 가능합니다. |
| **Back** | 백오더 수량. 음수이면 **빨간색**으로 표시됩니다. |
| **Sales Status** | 상품 상태 배지: `Original`(파란), `Custom`(초록), `Hold`(노란) |
| **Master SKU** | SKU명. 앞에 **긴급도 표시 점**이 있습니다 (🔴빨간=긴급, 🟡주황=주의, 🟢초록=정상). 클릭하면 전체 SKU명 복사 가능. |

---

### 3-2. 재고 컬럼 (Inventory)

| 컬럼 | 설명 |
|------|------|
| **West Stock** | 서부 창고 현재 재고량 |
| **East Stock** | 동부 창고 현재 재고량 |
| **Total Stock** | 전체 재고량 (굵은 글씨) |

---

### 3-3. 판매 컬럼 (West/East FBM Sales)

최근 기간별 실제 FBM 판매 수량입니다.  
서부(West)와 동부(East) 두 그룹으로 나뉩니다.

| 컬럼 | 설명 |
|------|------|
| 90D / 60D / **30D** / 15D / 7D | 각 기간 동안의 누적 판매량. **30D는 굵은 글씨** |
| Pre 30D | 이전 30일 판매량 (비교용) |

---

### 3-4. 일평균 판매량 컬럼 (W Avg / E Avg / FBA Avg)

세 가지 버전의 일평균 판매량을 보여줍니다.

| 컬럼 | 설명 |
|------|------|
| **이전 (Previous)** | 과거 기준 평균 |
| **실제 (Real)** | 실제 측정 평균 |
| **현재 (Current)** | 현재 적용 중인 평균 (색상으로 강도 표시) |

**현재 서부 일평균(W Avg 현재) 색상 기준**:
- **빨간색**: 하루 10개 이상 (속도 매우 빠름, 주의 필요)
- **주황색**: 하루 5~9개
- **일반색**: 하루 5개 미만

---

### 3-5. 30일 채널별 판매 (30D Sales)

| 컬럼 | 설명 |
|------|------|
| **W FBM 30D** | 서부 FBM 30일 판매 |
| **E FBM 30D** | 동부 FBM 30일 판매 |
| **FBA 30D** | FBA 30일 판매 |
| **Total 30D** | 전체 채널 합계 (굵은 글씨) |

---

### 3-6. 전체 채널 일평균 (Total Avg Daily)

| 컬럼 | 설명 |
|------|------|
| **T.Avg 이전** | 이전 전체 일평균 |
| **T.Avg 실제** | 실제 전체 일평균 (굵은 글씨) |
| **T.Avg 현재** | 현재 전체 일평균 (굵은 글씨) |

---

### 3-7. 입고·컨테이너·SOD 컬럼 (Inbound/SOD)

가장 중요한 컬럼 그룹입니다.

| 컬럼 | 설명 |
|------|------|
| **Inbound Qty** | 현재 입고 예정인 총 수량. 0이 아니면 **파란색**으로 표시됩니다. |
| **Containers List** | `컨테이너명 (수량), 컨테이너명 (수량)` 형식. 길면 클릭하면 전체 목록과 복사 버튼이 나타납니다. |
| **Next ETA** | 가장 빨리 도착하는 컨테이너의 예상 도착일. 지난 날짜는 **빨간색**, 14일 이내는 **주황색**, 그 외는 **파란색** |
| **S.O.D 품절예상일** | 현재 재고와 입고량으로 계산한 품절 예상일. **30일 이내: 빨간색**, **60일 이내: 주황색** |

> 💡 **S.O.D (Stock Out Date)**란?  
> "현재 재고 + 앞으로 들어올 재고"를 현재 판매 속도로 나누어 재고가 0이 되는 날짜를 계산한 값입니다.  
> SOD가 짧을수록 빨리 재고가 소진된다는 의미이므로 신속한 발주가 필요합니다.

## 4. 컨테이너 컬럼 상세 안내

**⊞ Columns 버튼 → 📋 Container 컬럼** 을 켜면 컨테이너별 상세 데이터가 표시됩니다.  
처음 활성화 시 데이터를 로딩합니다 (잠시 기다려 주세요).

### 4-1. 컨테이너 상태 색상

| 컨테이너 상태 | 표시 방식 | 설명 |
|-------------|-----------|------|
| **Baseline** | 🟢 녹색 배경 | 기준이 되는 현재 재고 상태 |
| **Draft** | ✏️ 회색 텍스트, 연필 아이콘 | 아직 출항하지 않은 발주 계획 단계 |
| **Shipped / Packing Received** | 파란 ETA 날짜 | 실제 선적된 활성 컨테이너 |

### 4-2. 컨테이너 서브 컬럼

각 컨테이너마다 아래 컬럼들이 나타납니다.

| 컬럼 | 설명 |
|------|------|
| **Con. Qty** | 해당 컨테이너의 입고 예정 수량. 활성 컨테이너는 **파란 밑줄**이 있으며 클릭해서 수정 가능 |
| **Remaining** | 해당 컨테이너 도착 후 판매하고 남을 예상 잔여 수량 |
| **Mistake** | 수량 계산 오류. **빨간색** 표시 |
| **Open Orders** | 현재 열려 있는 미이행 주문 수량 |
| **Avail Qty** | 이전 재고 + 이번 컨테이너 수량의 사용 가능량 |
| **Est. Sales** | 이전 컨테이너 도착~이번 컨테이너 도착 사이의 예상 판매량 |
| **Back Order** | 예상 판매를 충족 못 할 경우 발생하는 백오더. 빨간색 표시 |
| **Carry over** | 이번 컨테이너 도착 후 다음 컨테이너까지 남을 것으로 예상되는 수량 |
| **Inv. Life** | Carry over를 일평균 판매량으로 나눈 예상 재고 지속 기간(일). 🔴30일↓ / 🟡60일↓ / 🟣90일↑ |
| **EST. SOD** | 이 컨테이너 기준으로 계산한 품절 예상일. 🔴30일↓ / 🟡60일↓ |
| **Plan SOD** | 이 컨테이너 단독으로 계산한 계획 SOD |
| **CBM** | 이 컨테이너에서 이 SKU가 차지하는 총 부피(㎥) |

### 4-3. 컨테이너 ETA 수정

Draft 또는 Shipped 컨테이너 헤더에 있는 날짜를 클릭하면 **예상 도착일(ETA)을 수정**할 수 있습니다.  
날짜를 바꾸면 전체 컨테이너 체인의 예상 SOD가 즉시 재계산됩니다.

---

## 5. 직접 수정할 수 있는 데이터

### 5-1. CBM(부피) 수정

1. **CBM 컬럼** 셀 클릭 → 입력 필드로 바뀝니다
2. 새 값 입력 (소수점 4자리까지 가능)
3. **Enter** 키 또는 다른 곳 클릭 → 저장
4. **Esc** 키 → 취소

> 변경된 CBM은 즉시 반영되며 서버에 저장됩니다.

---

### 5-2. 컨테이너 입고 수량(Con. Qty) 수정

**파란 밑줄**이 있는 Con. Qty 셀만 수정 가능합니다 (Baseline은 수정 불가).

1. Con. Qty 셀 클릭 → 입력 필드로 바뀝니다
2. 수량 입력
   - **0 입력** → 해당 항목을 삭제합니다
   - **양수 입력** → 수량 저장
3. **Enter** 키 또는 다른 곳 클릭 → 저장
4. **Esc** 키 → 취소

> 수량을 수정하면 해당 컨테이너 이후의 Carry over, Inv. Life, Est. SOD가 모두 자동으로 재계산됩니다.

---

## 6. 컬럼 정렬

- 헤더 텍스트에 **▲▼ 화살표**가 나타나는 컬럼은 정렬이 가능합니다.
- **1번 클릭**: 오름차순 정렬 ▲
- **2번 클릭**: 내림차순 정렬 ▼
- 필터·카테고리를 변경하면 정렬이 초기화됩니다.

|------|------|------|

| Container Info. | 90px | 320px |
| Master SKU | 160px | 420px |
| Containers List | 120px | 420px |

조절 핸들을 **드래그**하여 너비를 변경합니다.  
설정은 **자동으로 저장**되며, 브라우저를 닫아도 유지됩니다.  
원래대로 되돌리려면 **⊞ Columns → Reset Column Widths** 버튼을 누르세요.

## 8. 긴급도(Urgency) 시스템 정리

대시보드 전체에서 사용되는 3단계 긴급도입니다.

| 등급 | 조건 | 시각적 표시 |
|------|------|------------|
| 🔴 **Critical (긴급)** | SOD ≤ 30일 | 행 배경 연분홍, SKU 점 빨간색, SOD 빨간 굵은 글씨 |
| 🟡 **Warning (주의)** | SOD ≤ 60일 | SKU 점 주황색, SOD 주황 글씨 |
| 🟢 **정상 (OK)** | SOD > 60일 | SKU 점 초록색, SOD 회색 글씨 |

> **SOD 계산 방식**: 현재 재고 + 선적 완료/포장 완료 상태의 컨테이너 입고 예정 수량을 현재 일평균 판매량으로 나눈 날짜 (Draft 컨테이너는 미포함)

## 11. 자주 사용하는 시나리오

### 📌 시나리오 1: 긴급 발주 필요한 SKU 찾기

1. **Status → Critical** 선택
2. SOD가 가장 짧은 순으로 정렬: **S.O.D 컬럼 헤더 클릭** ▲
3. 빨간색으로 표시된 SKU들을 확인하고 발주 조치

---

### 📌 시나리오 2: 특정 SKU의 컨테이너 입고 계획 확인

1. 검색창에 SKU명 입력
2. **⊞ Columns → 📋 Container 컬럼** 활성화 (로딩 대기)
3. 각 컨테이너의 Con. Qty, ETA, Carry over, EST. SOD 확인

---

### 📌 시나리오 3: 컨테이너 입고 수량 조정

1. 해당 SKU 행에서 수정하려는 컨테이너의 **Con. Qty** 셀 클릭
2. 새 수량 입력 후 **Enter**
3. Inv. Life, EST. SOD 등 연계 지표가 자동으로 재계산됩니다.

---

### 📌 시나리오 4: CBM 수정

1. 해당 SKU 행의 **CBM** 셀 클릭 (파란 밑줄 표시)

1. **⊞ Columns → Quick Preset → Compact** 선택
2. SKU, 재고, 전체 일평균, 입고량, SOD만 표시되는 간결한 화면

## 13. 자주 묻는 질문 (FAQ)

**Q. SOD가 "—" 로 표시되는데 뭔가요?**  
A. 품절 예상일을 계산할 수 없는 경우입니다. 주로 판매 데이터가 없거나 일평균이 0인 경우입니다.

**Q. 빨간색 행이 너무 많아요. 모두 Critical인가요?**  
A. 배경이 연분홍인 행은 SOD가 30일 이내인 긴급 SKU입니다. Status 필터에서 "Critical"을 선택하면 해당 행만 볼 수 있습니다.

**Q. 컨테이너 컬럼이 없는데 어디서 켜나요?**  
A. 상단의 **⊞ Columns → Column Groups → 📋 Container 컬럼** 체크박스를 켜세요. 처음 켤 때는 데이터를 로딩합니다.

**Q. 수량을 수정했는데 반영이 안 돼요.**  
A. 편집 중 **Enter** 키를 눌러야 저장됩니다. Esc를 누르면 취소됩니다.

**Q. Draft 컨테이너는 왜 SOD 계산에 포함 안 되나요?**  
A. Draft는 아직 확정되지 않은 발주 계획 단계입니다. 출항 완료(Shipped) 또는 패킹리스트 수령(Packing Received) 상태가 된 컨테이너만 실제 입고로 인정하여 SOD를 계산합니다.

**Q. 데이터가 오래된 것 같아요.**  
A. 우측의 **Sync** 버튼을 눌러 최신 데이터를 불러오세요. 마지막 동기화 시각은 버튼 왼쪽에 표시됩니다.

**Q. 컬럼이 너무 많아서 어디를 봐야 할지 모르겠어요.**  
A. **⊞ Columns → Core** 프리셋으로 시작하세요. 핵심 지표만 남기고 나머지는 숨깁니다.
