시작하기

Demand Pilot의 전체 메뉴 구조와 사용 흐름을 안내합니다.

Demand Pilot은 e-commerce 상품의 재고 현황 파악 → 발주 계획 → 컨테이너 입고 관리를 하나의 화면에서 처리하는 운영 도구입니다.

메뉴 구성

1

수요 계획

SKU별 재고 현황과 컨테이너 발주 계획을 한눈에
2

SKU 계획

개별 SKU의 판매 추이와 발주 권장량 분석
3

컨테이너 계획

컨테이너 등록 · 패킹 리스트 · 상태 관리
4

컨테이너 일정

ETA 기준 간트 보기 · 입고 일정 한눈에 보기
5

가용 재고

잔여 재고 · 오류 목록 관리
6

SKU 마스터

SKU별 CBM · MOQ · 케이스 수량 기준 데이터 관리

기본 사용 흐름

1

SKU 마스터 확인

CBM(부피)·MOQ·발주 배수가 정확히 등록되어 있어야 발주 계산이 올바르게 동작합니다. 신규 SKU 추가 시 가장 먼저 확인하세요.

2

컨테이너 계획에서 컨테이너 등록

입고 예정 컨테이너를 등록하고 ETA(도착 예정일)를 입력합니다. SKU별 수량을 패킹 리스트로 관리합니다.

3

수요 계획으로 발주 계획

재고 현황과 컨테이너 체인을 확인하며 자동 계산 버튼으로 최적 발주 수량을 산출합니다.

4

가용 재고에서 여유 재고 배정

창고에 이미 있는 잔여 재고 또는 오류 재고를 컨테이너에 배정해 입고량을 조정합니다.

5

SKU 계획으로 세부 분석

판매 추이가 궁금한 SKU를 선택해 판매 추이와 재고 소진일을 확인합니다.

상단 내비게이션 구성

커맨드 센터

로그인 직후 나타나는 홈 화면으로, 재고 상황·컨테이너 입고·판매 추세를 한눈에 파악합니다.

커맨드 센터는 Demand Pilot의 홈 대시보드입니다. 별도의 클릭 없이 로그인하자마자 현재 운영 상태를 즉시 파악할 수 있도록 설계되어 있습니다. 상단에서 제품 카테고리를 전환하면 각 카테고리별 KPI가 실시간으로 업데이트됩니다.

▲ 커맨드 센터 — 로그인 후 홈 화면 전체 구성

카테고리 탭

화면 상단의 탭으로 제품군을 전환합니다. KPI 카드와 위험 SKU 목록, 재고 분포 도넛 차트가 선택한 카테고리 기준으로 즉시 업데이트됩니다.

탭표시 범위
시트 커버SC 카테고리 SKU 전체
카 커버CC 카테고리 SKU 전체
플로어 매트FM 카테고리 SKU 전체

① KPI 카드 (5개)

선택한 카테고리의 핵심 지표를 한눈에 보여주는 5개의 클릭 가능한 카드입니다. 각 카드를 클릭하면 해당 조건으로 필터링된 수요 계획 또는 컨테이너 일정 화면으로 바로 이동합니다.

▲ KPI 카드 — 카테고리 탭과 5개 지표 카드
카드의미클릭 시 이동
위험 SKU SOD(품절예상일)가 30일 이내인 SKU 수. 수요 계획 → Critical 필터 적용
품절 예상 SOD가 31~60일 이내인 SKU 수. 주의 관찰이 필요한 SKU입니다. 수요 계획 → Warning 필터 적용
과잉 재고 SOD가 180일 초과인 SKU 수. 과잉 재고로 자본이 묶여 있는 상태입니다. 수요 계획 → Overstock 필터 적용
긴급 발주 위험(Critical) 상태이면서 입고 예정 컨테이너가 없는 SKU 수. 즉시 발주가 필요합니다. 수요 계획 → Critical + 입고 없음 필터
지연 컨테이너 ETA가 이미 지났는데 아직 입고 완료되지 않은 컨테이너 수. 전체 카테고리 기준입니다. 컨테이너 일정 화면

② 3패널 현황

KPI 카드 아래에는 세 개의 패널이 나란히 표시됩니다. 각 패널의 "전체 보기 →" 링크를 클릭하면 상세 화면으로 이동합니다.

▲ 3패널 — 좌: 위험 SKU TOP 10 / 중: 입고 예정 컨테이너 / 우: 재고 분포

🔴 위험 SKU TOP 10

선택한 카테고리에서 SOD가 가장 짧은 SKU를 최대 10개 보여줍니다. SKU명을 클릭하면 SKU 계획 화면으로 바로 이동합니다. 각 행에 일평균 판매량(Daily Avg)이 함께 표시됩니다.

🚢 입고 예정 컨테이너

ETA 순으로 정렬된 예정 컨테이너 목록입니다. 컨테이너명 클릭 시 컨테이너 일정으로 이동합니다. ETA가 이미 지난 항목은 빨간색으로 표시됩니다. 상태 배지(Draft / Packing / Shipped / Complete)로 진행 단계를 한눈에 확인할 수 있습니다.

🍩 재고 분포 (Stock Health)

선택 카테고리의 SKU를 SOD 구간별로 나누어 도넛 차트로 시각화합니다.

  • ■ 위험 — SOD ≤ 30일
  • ■ 품절 예상 — 31~60일
  • ■ 정상 — 61~180일
  • ■ 과잉 재고 — 181일+

③ 판매 추세 차트

전체 채널의 판매 추세를 차트로 표시합니다. 차트 영역 전체를 클릭하면 주문 내역 화면으로 이동합니다.

▲ 판매 추세 — 기간·뷰·차트 유형 선택 가능
컨트롤설명
기간 선택 최근 7일 / 30일 / 90일 / 6개월 / 1년 / 올해 누계(YTD) 중 선택합니다. 선택한 기간과 직전 동일 기간을 자동으로 비교합니다.
매출 / 주문 토글 Y축을 매출(Revenue, $) 또는 주문 수량(Orders)으로 전환합니다.
차트 유형 영역(Area) · 선(Line) · 막대(Bar) 중 선택합니다. 원하는 시각화 스타일로 자유롭게 변경할 수 있습니다.

차트 오른쪽의 판매 요약 카드에는 선택 기간의 총 주문 수, 총 매출, 전기 대비 성장률(%)이 표시됩니다.

⑤ Complete 상태와 Demand Planning 표시 규칙

⚠️
Complete 상태 컨테이너는 Demand Planning에 표시되지 않습니다.
컨테이너를 Complete로 변경하는 순간, 해당 컨테이너는 Demand Planning의 컨테이너 열 목록에서 즉시 사라지고, SKU별 발주 계획(Con. Qty) 계산에서도 제외됩니다.

시스템이 이렇게 동작하는 이유는 입고가 완료된 재고는 이미 창고 실재고(Current Stock)에 반영되었기 때문입니다. 이후 발주 계획은 갱신된 실재고 수치를 기준으로 다시 계산하면 됩니다.

🚫 Demand Planning에서 제외되는 조건

  • 상태 = Complete
  • 컨테이너 열이 사라지고 Con. Qty 계산에서 제외
  • 자동계산 버튼(📅 / 📊) 결과에도 미반영
  • SKU Planning > 재고 및 입고 탭의 입고 컨테이너 목록에도 미표시

✅ Demand Planning에 표시되는 조건

  • 상태 = Final List Sent
  • 상태 = Shipped (Packing List Rcvd)
  • 상태 = Draft (Draft 포함 토글 ON 시에만)

상태 전환 시 Demand Planning 변화 흐름

Draft → Final List Sent → Shipped (Packing List Rcvd) → Complete ❌

Complete로 변경되는 순간 Demand Planning 컨테이너 목록에서 제거됩니다.

상황권장 조치
실제 입고가 완료된 컨테이너 Complete로 변경 → 창고 재고에 반영되었으므로 Demand Planning에서 자동 제외. 이후 재고 현황은 Demand Planning의 현재 재고(Stock) 수치로 확인.
아직 창고 입고 전인데 Complete로 잘못 변경한 경우 Container Planning에서 해당 컨테이너를 찾아 상태를 Shipped (Packing List Rcvd)로 되돌리면 Demand Planning에 즉시 다시 표시됩니다.
Complete 컨테이너 내역을 다시 보고 싶은 경우 Container Planning 페이지에서 직접 조회합니다. Demand Planning과 SKU Planning에서는 Complete 컨테이너가 표시되지 않습니다.
✅
올바른 Complete 처리 순서:
  1. 실물 입고 확인 후 Container Planning에서 해당 컨테이너 클릭
  2. 상태를 Complete로 변경
  3. 외부 재고 DB 동기화 후 Demand Planning에서 🔄 Sync 클릭
  4. 현재 재고(Stock) 수치가 갱신된 것을 확인하고 발주 계획을 재검토
⚠️
CBM 초과 주의: 화면 상단에 현재 CBM 사용률이 표시됩니다. 80 CBM을 초과하지 않도록 주의하세요.

Basic Workflow

1

Verify SKU Master

CBM (volume), MOQ, and order multiples must be correctly registered for purchase calculations to work. Always check this first when adding a new SKU.

2

Register Containers in Container Planning

Add inbound containers and enter the ETA (estimated arrival date). Manage SKU quantities as a packing list.

3

Plan Orders in Demand Planning

Review inventory status and container chains, then use auto-fill buttons to calculate optimal order quantities.

4

Allocate Surplus in Available Stock

Assign surplus inventory already in the warehouse (Remaining / Mistake Order) to containers to adjust inbound quantities.

5

Analyze Details in SKU Planning

Select a SKU to review its sales trends and estimated sell-out date.

Forecast Model Guide

Use Auto unless you have a specific reason to choose otherwise.

🤖 Auto (Recommended)

Cross-validation automatically picks the most accurate model for this SKU.

📊 AutoARIMA

Automatically identifies trends and seasonality. Best for products with clear annual seasonal patterns.

📈 AutoETS

Exponential smoothing — strong at smooth, gradual trends up or down.

📉 AutoTheta

Good at capturing long-term trends. Suited for steadily growing or declining products.

📋 Window Avg

Uses the average of the most recent N weeks. Best when sales are flat with no trend.

📋 Historic Avg

Uses the average of all history. Best when long-run average is stable.

🔄 Seasonal Naïve

Repeats the same week from last year. Best for products with strong annual seasonality.

➡️ Naïve

Repeats last week's sales. Used for highly erratic products or as a baseline comparison.

⑤ Complete Status and Demand Planning Display Rules

⚠️
Complete containers are not shown in Demand Planning.
The moment a container is changed to Complete, it immediately disappears from the Demand Planning container column list and is excluded from per-SKU Con. Qty calculations.

This behavior exists because inventory from a received container is already reflected in the warehouse's live stock figures (Current Stock). Subsequent order planning should be recalculated against the updated live stock numbers.

🚫 Conditions that Exclude from Demand Planning

  • Status = Complete
  • Container column disappears; Con. Qty excluded from calculations
  • Not reflected in Auto-Fill results (📅 / 📊)
  • Not shown in SKU Planning › Inventory & Inbound container list

✅ Conditions that Show in Demand Planning

  • Status = Final List Sent
  • Status = Shipped (Packing List Rcvd)
  • Status = Draft (only when Include Draft toggle is ON)

Demand Planning Changes When Status Transitions

Draft → Final List Sent → Shipped (Packing List Rcvd) → Complete ❌

The moment status changes to Complete, the container is removed from the Demand Planning container list.

SituationRecommended Action
Container has actually been receivedChange to Complete → Excluded from Demand Planning automatically. Verify updated stock in Demand Planning's Current Stock column.
Accidentally set to Complete before actual receiptFind the container in Container Planning and revert to Shipped (Packing List Rcvd) — it will immediately reappear in Demand Planning.
Need to review a Complete container's detailsView it directly in Container Planning. Complete containers are not shown in Demand Planning or SKU Planning.

  1. Confirm physical receipt, then click the container in Container Planning

Glossary

Common terms used in Demand Pilot.

TermDescription
CBMCubic Meter. Volume per box (m³). The basis for calculating container load capacity.
MOQMinimum Order Quantity. The minimum quantity that can be ordered in one purchase.
ETAEstimated Time of Arrival. The expected arrival date of a container.
SODSell-Out Date. The estimated date when current inventory will be fully depleted.
INV LifeInventory Life. How many days current inventory will last (Stock ÷ Daily avg sales).
Back OrderUnfulfilled demand resulting from insufficient inventory. A negative stock state.
CarryoverResidual inventory remaining after a container's arrival point.
Transit StockInventory currently in transit (not yet received into the warehouse).
Con. QtyContainer Quantity. The order quantity to be loaded into a container.
Gradient TierThe tiered target-days system used by the 📊 auto-fill button. Each tier pairs a minimum daily sales threshold (Min Avg Sales) with a Target Days value. Default: ≥10/day → 90d, ≥5/day → 80d, ≥3/day → 70d, others → 60d. Fully configurable via the tier dialog.
Remaining ListNormal finished-goods inventory waiting in the warehouse for inbound after production.
Mistake OrderSurplus inventory resulting from order errors or cancellations.
