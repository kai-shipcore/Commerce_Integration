시작하기

Demand Pilot의 전체 메뉴 구조와 사용 흐름을 안내합니다.

Demand Pilot은 아마존 기반 시트 커버 · 카 커버 · 플로어 매트 상품의 재고 현황 파악 → 발주 계획 → 컨테이너 입고 관리를 하나의 화면에서 처리하는 운영 도구입니다.

메뉴 구성

1

Demand Planning

SKU별 재고 현황과 컨테이너 발주 계획을 한눈에
2

SKU Planning

개별 SKU의 판매 추이와 발주 권장량 분석
3

Container Planning

컨테이너 등록 · 패킹 리스트 · 상태 관리
4

Available Stock

여유 재고 · 미스테이크 오더 목록 관리
5

SKU Master

SKU별 CBM · MOQ · 케이스 수량 기준 데이터 관리

기본 사용 흐름

1

SKU Master 확인

CBM(부피)·MOQ·발주 배수가 정확히 등록되어 있어야 발주 계산이 올바르게 동작합니다. 신규 SKU 추가 시 가장 먼저 확인하세요.

2

Container Planning에서 컨테이너 등록

입고 예정 컨테이너를 등록하고 ETA(도착 예정일)를 입력합니다. SKU별 수량을 패킹 리스트로 관리합니다.

3

Demand Planning으로 발주 계획

재고 현황과 컨테이너 체인을 확인하며 자동 계산 버튼으로 최적 발주 수량을 산출합니다.

4

Available Stock에서 여유 재고 배정

창고에 이미 있는 여유 재고(Remaining / Mistake Order)를 컨테이너에 배정해 입고량을 조정합니다.

5

SKU Planning으로 세부 분석

판매 트렌드가 궁금한 SKU를 선택해 판매 추이와 재고 소진일을 확인합니다.

Demand Planning

전체 SKU의 재고 현황과 컨테이너 발주 계획을 통합 관리하는 메인 화면입니다.

▲ Demand Planning – 메인 화면 (재고 현황 + 컨테이너 체인)

① 상단 필터 & 검색

컨트롤설명
제품 선택 드롭다운Seat Cover / Car Cover / Floor Mat / Accessories 중 하나를 선택해 해당 카테고리 SKU만 표시합니다. 선택에 따라 컨테이너 목록도 자동으로 필터링됩니다.
제품 타입Original / Custom으로 SKU를 추가 분류합니다.
긴급도 필터Critical Warning BackOrder 상태의 SKU만 추려볼 때 사용합니다.
🔍 검색창SKU 이름 또는 컨테이너 번호로 실시간 필터링합니다.
Con Qty 필터발주 수량이 입력된 SKU / 비어 있는 SKU만 표시합니다.
날짜 (As of)특정 날짜 기준 과거 재고 현황을 조회할 때 사용합니다. 기본값은 오늘입니다.
Sync 버튼서버에서 최신 재고 데이터를 다시 불러옵니다.
Export 버튼현재 필터가 적용된 데이터를 CSV로 내보냅니다.
Import 버튼Transit Stock(해상 운송 중 재고)을 엑셀로 일괄 업로드합니다.

SKU 파트 필터 (Seat Cover 전용)

Seat Cover를 선택하면 추가로 Seat / No. / Color / Tone 4가지 필터가 나타납니다. 각 드롭다운에서 원하는 값을 복수 선택해 특정 사양의 SKU만 볼 수 있습니다.

② 테이블 컬럼 구성

Columns 버튼을 눌러 표시할 컬럼 그룹을 선택할 수 있습니다.

📌 고정 컬럼

  • SKU 이름
  • West / East / Total 재고
  • Transit Stock (운송 중)
  • Back 수량, SOD(재고 소진일)

📈 판매 컬럼

  • W30D / E30D – 서/동 30일 판매
  • Total30D – 총 30일 판매
  • Total Avg Curr – 현재 일평균 판매
  • Inbound – 입고 예정 수량

🚢 컨테이너 컬럼

  • Con. Qty – 발주 수량 (편집 가능)
  • Avail Qty – 컨테이너 도착 후 가용 재고
  • Est. Sales – 예상 판매량
  • Back Order – 예상 재고 부족량
  • Rem. Qty – 잔여 재고
  • Inv. Life – 재고 유지 일수
  • EST. SOD – 예상 재고 소진일

③ Con. Qty 자동계산 버튼

각 컨테이너 헤더에는 두 가지 자동계산 버튼이 있습니다.

UI 배치: 컨테이너 헤더에는 ETA 날짜 → ⟳ → 90(숫자 입력) → ⟳₂ → ⟳₃ 순서로 배치되어 있습니다.
숫자 입력칸(90)은 ⟳₂ 전용이며, ⟳와 ⟳₃은 이 값을 사용하지 않습니다.
버튼목표 일수 결정 방식언제 사용?
⟳ 자동 결정 (입력값 불필요) — 컨테이너 CBM 용량(기본 80 CBM)을 최대한 채우는 최적 Inv. Life를 이진탐색으로 계산. 판매량 티어별 보너스(±일수) 자동 적용. 컨테이너 공간을 최대 활용해 최적 발주 구성이 필요할 때
90 입력 후 ⟳₂ 숫자 입력칸의 목표 일수 사용 — 발주량 = ROUNDUP((일판매 × 목표일 + BackOrder − 잔여재고) ÷ step) × step. 모든 SKU에 동일한 목표 일수 적용. 특정 재고 유지 일수(예: 90일)를 모든 SKU에 동일하게 적용할 때
⟳₃ 다이얼로그에서 구간별 목표 설정 (입력값 불필요) — 클릭 시 다이얼로그가 열리며 일판매량 구간별로 목표 일수를 다르게 설정. 기본: ≥10개→90일, ≥5개→80일, ≥3개→70일, 나머지→60일. 판매량 많은 SKU는 길게, 적은 SKU는 짧게 차등 적용할 때
💡
저장 전 미리보기: 세 버튼 모두 클릭 시 즉시 수량이 반영되지만 DB에는 저장되지 않습니다. 확인 후 💾 Save 버튼을 눌러야 최종 저장됩니다. 되돌리려면 ↺ Reset을 누르세요.

⟳₂ 버튼 목표 일수 변경 방법

1

⟳₂ 버튼 왼쪽의 숫자 입력칸 클릭

기본값은 90입니다. 클릭 후 원하는 일수를 입력합니다 (예: 60, 120).

2

⟳₂ 버튼 클릭

입력한 일수를 기준으로 모든 SKU의 발주 수량이 다시 계산됩니다.

3

💾 Save 클릭

결과가 맞으면 Save를 눌러 DB에 저장합니다.

⟳₃ 버튼 구간별 목표 설정 방법

1

⟳₃ 버튼 클릭

다이얼로그가 열리며 현재 구간 설정이 표시됩니다.

2

구간 설정 확인 및 수정

Min Avg Sales(일판매 최솟값)와 Target Days(목표 재고 일수)를 원하는 값으로 수정합니다. +로 구간 추가, ×로 구간 삭제 가능합니다.

기본 구간Min Avg SalesTarget Days
구간 1≥ 10개/일90일
구간 2≥ 5개/일80일
구간 3≥ 3개/일70일
구간 4 (기본)≥ 0개/일60일
3

Apply 클릭

각 SKU의 일판매량에 해당하는 구간의 목표 일수로 발주 수량이 계산됩니다.

4

💾 Save 클릭

결과가 맞으면 Save를 눌러 DB에 저장합니다.

✅
⟳₃ 구간 설정은 세션 동안 유지됩니다. 한 번 설정한 구간은 페이지를 벗어나기 전까지 다른 컨테이너에 ⟳₃를 눌러도 동일하게 적용됩니다. 컨테이너마다 다른 구간을 쓰려면 매번 다이얼로그에서 수정하세요.

④ 컨테이너 표시 조건

Demand Planning의 컨테이너 열은 모든 상태의 컨테이너가 표시되지 않습니다. 아래 규칙에 따라 필터링됩니다.

컨테이너 상태Demand Planning 표시비고
Final List Sent✅ 항상 표시공장 발송 완료 상태
Packing List Rcvd✅ 항상 표시패킹 리스트 수령 상태
Draft⬜ Draft 포함 토글 ON 시만 표시상단 필터에서 수동 활성화 필요
Complete❌ 표시 안 됨입고 완료 처리된 컨테이너는 계획 대상에서 제외
💡
Complete 컨테이너가 목록에 없다면? Container Planning 페이지에서 해당 컨테이너의 상태를 확인하세요. 상태가 Complete이면 입고 완료 처리된 것이며, Demand Planning과 SKU Planning 재고 및 입고 탭 모두에서 표시되지 않습니다. 자세한 내용은 Container Planning > Complete 상태 설명을 참고하세요.

⑤ Gradient(티어) 설정

판매량이 높은 SKU에는 더 긴 재고 목표를 적용하는 Gradient 티어가 자동 적용됩니다. 하단 상태바의 Gradient 영역에서 기준값을 직접 수정할 수 있습니다.

티어일판매 기준 (SC 기준)보너스 일수
TOP≥ 18개/일+15일
A≥ 8개/일+5일
B≥ 3개/일±0일
C≥ 0.3개/일−5일
D≥ 0.1개/일−10일
⚠️
계절 배수: 컨테이너 ETA 월에 따라 일판매에 계절 배수가 자동 적용됩니다 (11월 ×1.25, 12월 ×1.3). 하단 상태바의 Seasonal에서 월별 배수를 수정할 수 있습니다.

SKU Planning

개별 SKU의 판매 추이, 재고·입고 현황, 발주 권장량을 한 화면에서 상세 분석합니다.

▲ SKU Planning – 좌측 SKU 브라우저 + 우측 3탭 상세 분석

화면 전체 구성

📋 좌측 패널 – SKU 브라우저 (580px)

  • SC / CC / FM 제품 탭 전환 (건수 뱃지 표시)
  • 검색창: SKU명·컨테이너 번호·판매 상태 실시간 필터
  • 긴급도별 필터 뱃지 (All / Critical / Watch / High / Low / Order Required)
  • SKU 목록: 재고·일판매량·SOD·권장수량 열람 및 정렬

📊 우측 패널 – SKU 상세 분석

  • SKU 헤더: 이름·제품명·상태·MOQ·CBM·다음 ETA
  • KPI Strip: 5개 핵심 지표 카드
  • 판매 분석 탭: 기간별 판매 추이 차트
  • 재고 및 입고 탭: 재고 커버리지·입고 컨테이너 목록
  • 컨테이너 추천 탭: 권장 발주 수량 + What-If 시뮬레이션

상단 전역 컨트롤

컨트롤 (화면 레이블)설명
제품 탭 (SC / CC / FM)Seat Cover · Car Cover · Floor Mat 중 하나를 선택합니다. 탭 우측에 해당 제품의 총 SKU 수가 표시됩니다.
Sales 있는 SKU / 전체 SKU토글. Sales 있는 SKU: 최근 판매 실적이 있는 SKU만 표시. 전체 SKU: 판매 이력 없는 SKU까지 모두 표시.
Active 컨테이너 / Draft 포함토글. Active 컨테이너: 확정된 컨테이너 기준 재고 계산. Draft 포함: 초안(Draft) 컨테이너 입고량까지 포함하여 계산.
목표 재고일 (1–365)컨테이너 추천 탭의 목표 재고 일수를 설정합니다. 이 값이 커질수록 권장 발주 수량이 늘어납니다. KPI의 재고 유지일 기준이 됩니다.
언어 전환 (KO / EN)화면 레이블을 한국어·영어로 전환합니다. localStorage에 저장됩니다.
🔄 동기화서버에서 최신 수요 계획 데이터를 다시 불러옵니다.

① 좌측 SKU 브라우저 패널

검색 및 필터 뱃지

검색창에 SKU 이름, 컨테이너 번호, 또는 판매 상태 일부를 입력하면 목록이 실시간으로 좁혀집니다. 검색창 아래에는 긴급도별 필터 뱃지가 있어 클릭 한 번으로 특정 상태의 SKU만 볼 수 있습니다.

필터 뱃지조건

Critical긴급도 = Critical인 SKU (재고 소진 임박)
Watch긴급도 = Watch인 SKU (재고 주의 필요)
High긴급도 = Healthy인 SKU (재고 양호)
LowBack Order(재고 부족) 수치가 음수인 SKU

컬럼 헤더내용

재고현재 총 재고 수량 (West + East). 클릭 시 오름차순/내림차순 정렬.

② KPI 요약 카드 (5개)

SKU를 선택하면 탭 위에 5가지 핵심 지표가 카드 형태로 표시됩니다.

📦 현재 재고

현재 창고 총 재고. 하단에 West / East 창고별 분리 수량이 표시됩니다.

🚢 입고

입고 예정 총 수량. 부제목으로 Draft 포함 여부와 가장 가까운 다음 ETA 날짜가 표시됩니다. 입고 예정이 없으면 "활성 입고 없음" 표시.

📈 예상 재고

현재 재고 + 입고 예정 + 백오더 영향을 합산한 예상 보유 재고. 부제목에 SKU의 CBM/개 값이 표시됩니다.

📊 일평균 판매

현재 일평균 판매 속도. 부제목에 최근 30일 총 판매량(단위: 개)이 표시됩니다.

⏱️ 재고 유지일

예상 재고 유지 일수와 예상 소진일(SOD). 입고가 있을 경우 "현재 재고만" 일수(입고 제외 시 몇 일인지)도 긴급도 색상으로 표시합니다.

⚠️

권장 수량 산출 근거 (4줄 설명)

권장 박스 우측에 4가지 계산 근거가 불릿 형태로 표시됩니다.

① 목표 재고

목표 재고 − 예상 재고, 발주 배수 단위로 올림(ROUNDUP)

④ 속도 컨텍스트

7D 평균 vs 30D 평균 비교 및 변화율(%). 최근 판매가 가속/감속 중인지 표시.

② 상태별 주의사항

상태편집 가능 항목Demand Planning 표시 여부비고

③ Complete 상태와 Demand Planning 표시 규칙

⚠️
Complete 상태 컨테이너는 Demand Planning에 표시되지 않습니다.
컨테이너를 Complete로 변경하는 순간, 해당 컨테이너는 Demand Planning의 컨테이너 열 목록에서 즉시 사라지고, SKU별 발주 계획(Con. Qty) 계산에서도 제외됩니다.

시스템이 이렇게 동작하는 이유는 입고가 완료된 재고는 이미 창고 실재고(Current Stock)에 반영되었기 때문입니다. 이후 발주 계획은 갱신된 실재고 수치를 기준으로 다시 계산하면 됩니다.

🚫 Demand Planning에서 제외되는 조건

  • 상태 = Complete
  • 컨테이너 열이 사라지고 Con. Qty 계산에서 제외
  • 자동계산 버튼(⟳ / ⟳₂ / ⟳₃) 결과에도 미반영
  • SKU Planning > 재고 및 입고 탭의 입고 컨테이너 목록에도 미표시

✅ Demand Planning에 표시되는 조건

  • 상태 = Final List Sent
  • 상태 = Packing List Rcvd
  • 상태 = Draft (Draft 포함 토글 ON 시에만)

상태 전환 시 Demand Planning 변화 흐름

Draft → Final List Sent → Packing List Rcvd → Complete ❌

Complete로 변경되는 순간 Demand Planning 컨테이너 목록에서 제거됩니다.

상황권장 조치
실제 입고가 완료된 컨테이너 Complete로 변경 → 창고 재고에 반영되었으므로 Demand Planning에서 자동 제외. 이후 재고 현황은 Demand Planning의 현재 재고(Stock) 수치로 확인.
아직 창고 입고 전인데 Complete로 잘못 변경한 경우 Container Planning에서 해당 컨테이너를 찾아 상태를 Packing List Rcvd로 되돌리면 Demand Planning에 즉시 다시 표시됩니다.
Complete 컨테이너 내역을 다시 보고 싶은 경우 Container Planning 페이지에서 직접 조회합니다. Demand Planning과 SKU Planning에서는 Complete 컨테이너가 표시되지 않습니다.
✅
올바른 Complete 처리 순서:
  1. 실물 입고 확인 후 Container Planning에서 해당 컨테이너 클릭
  2. 상태를 Complete로 변경
  3. 외부 재고 DB 동기화 후 Demand Planning에서 🔄 Sync 클릭
  4. 현재 재고(Stock) 수치가 갱신된 것을 확인하고 발주 계획을 재검토
💡
패킹 리스트 내보내기: 컨테이너 상세 화면에서 Export Excel을 클릭하면 패킹 리스트를 엑셀 파일로 다운로드할 수 있습니다.

SKU별 CBM(부피)·MOQ(최소 발주)·발주 배수·케이스 수량 등 기준 데이터를 관리합니다.

▲ SKU Master – 전체 SKU 목록 및 편집 화면

주요 컬럼 설명

항목설명예시
Master SKU고유 SKU 식별자CA-SC-10-F-10-BK-1TO
CBM / Unit박스 1개당 부피(㎥). 발주 계산의 핵심 값.0.042500
MOQ최소 발주 수량. 이 수량 미만으로 발주 불가.10
Order Multiple발주 수량의 배수 단위 (예: 5 → 5, 10, 15, 20…)5
Case Qty박스 1개에 들어가는 제품 수량1
StatusActive / Inactive – 비활성화 시 발주 계산에서 제외

SKU 정보 수정하기

1

검색으로 SKU 찾기

상단 검색창에 SKU 이름 일부를 입력하거나 제품 타입 드롭다운으로 범위를 좁힙니다.

2

Edit 버튼 클릭

수정할 SKU 행의 Edit 버튼을 클릭하면 CBM·MOQ·배수·케이스 수량 필드가 편집 가능해집니다.

3

값 수정 후 Done 클릭

수정 완료 후 Done을 클릭하면 자동으로 저장됩니다.

💡
일괄 CBM 업데이트: Import Excel로 여러 SKU의 CBM을 한번에 업데이트할 수 있습니다.
🔄
재고 동기화: Sync Inventory를 누르면 외부 재고 DB에서 최신 데이터를 가져옵니다.
📖

용어 설명

Demand Pilot에서 자주 사용하는 용어 정리입니다.

용어설명
CBMCubic Meter. 박스 1개의 부피(㎥). 컨테이너 적재량 계산의 기준.
MOQMinimum Order Quantity. 한 번에 발주 가능한 최소 수량.
ETAEstimated Time of Arrival. 컨테이너 도착 예정일.
SODSell-Out Date. 현재 재고가 모두 소진되는 예상 날짜.
INV LifeInventory Life. 현재 재고가 몇 일간 유지되는지 나타내는 지표 (재고 ÷ 일평균 판매).
Back Order재고 부족으로 발생한 미처리 수요. 음수 재고 상태.
Carryover컨테이너 도착 이후 남는 잔여 재고.
Transit Stock현재 해상 운송 중인 재고 (아직 창고에 미입고).
Con. QtyContainer Quantity. 해당 컨테이너에 담을 발주 수량.
Gradient Tier일판매량 구간별 재고 목표 일수 보너스 테이블. TOP·A·B·C·D 5단계.
Remaining List생산 완료 후 창고에서 입고 대기 중인 정상 재고 목록.
Mistake Order발주 오류·취소 등으로 발생한 잉여 재고 목록.
