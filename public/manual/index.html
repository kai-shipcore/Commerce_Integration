시작하기

Demand Pilot의 전체 메뉴 구조와 사용 흐름을 안내합니다.

Demand Pilot은 e-commerce 상품의 재고 현황 파악 → 발주 계획 → 컨테이너 입고 관리를 하나의 화면에서 처리하는 운영 도구입니다.

메뉴 구성

1

수요 계획

SKU별 재고 현황과 컨테이너 발주 계획을 한눈에
2

SKU 계획

개별 SKU의 판매 추이와 발주 권장량 분석
3

컨테이너 계획

컨테이너 등록 · 패킹 리스트 · 상태 관리
4

컨테이너 일정

Gantt · 월별 달력 · 스케줄 — 3가지 뷰로 입고 일정 한눈에 보기
5

가용 재고

잔여 재고 · 오류 목록 관리
6

SKU 마스터

SKU별 CBM · MOQ · 케이스 수량 기준 데이터 관리

기본 사용 흐름

1

SKU 마스터 확인

CBM(부피)·MOQ·발주 배수가 정확히 등록되어 있어야 발주 계산이 올바르게 동작합니다. 신규 SKU 추가 시 가장 먼저 확인하세요.

2

컨테이너 계획에서 컨테이너 등록

입고 예정 컨테이너를 등록하고 ETA(도착 예정일)를 입력합니다. SKU별 수량을 패킹 리스트로 관리합니다.

3

수요 계획으로 발주 계획

재고 현황과 컨테이너 체인을 확인하며 자동 계산 버튼으로 최적 발주 수량을 산출합니다.

4

가용 재고에서 여유 재고 배정

창고에 이미 있는 잔여 재고 또는 오류 재고를 컨테이너에 배정해 입고량을 조정합니다.

5

SKU 계획으로 세부 분석

판매 추이가 궁금한 SKU를 선택해 판매 추이와 재고 소진일을 확인합니다.

상단 내비게이션 구성

커맨드 센터

로그인 직후 나타나는 홈 화면으로, 재고 상황·컨테이너 입고·판매 추세를 한눈에 파악합니다.

커맨드 센터는 Demand Pilot의 홈 대시보드입니다. 별도의 클릭 없이 로그인하자마자 현재 운영 상태를 즉시 파악할 수 있도록 설계되어 있습니다. 상단에서 제품 카테고리를 전환하면 각 카테고리별 KPI가 실시간으로 업데이트됩니다.

▲ 커맨드 센터 — 로그인 후 홈 화면 전체 구성

카테고리 탭

화면 상단의 탭으로 제품군을 전환합니다. KPI 카드와 위험 SKU 목록, 재고 분포 도넛 차트가 선택한 카테고리 기준으로 즉시 업데이트됩니다.

탭표시 범위
시트 커버SC 카테고리 SKU 전체
카 커버CC 카테고리 SKU 전체
플로어 매트FM 카테고리 SKU 전체

① KPI 카드 (5개)

선택한 카테고리의 핵심 지표를 한눈에 보여주는 5개의 클릭 가능한 카드입니다. 각 카드를 클릭하면 해당 조건으로 필터링된 수요 계획 또는 컨테이너 일정 화면으로 바로 이동합니다.

▲ KPI 카드 — 카테고리 탭과 5개 지표 카드
카드의미클릭 시 이동
위험 SKU SOD(품절예상일)가 30일 이내인 SKU 수. 수요 계획 → Critical 필터 적용
품절 예상 SOD가 31~60일 이내인 SKU 수. 주의 관찰이 필요한 SKU입니다. 수요 계획 → Warning 필터 적용
과잉 재고 SOD가 180일 초과인 SKU 수. 과잉 재고로 자본이 묶여 있는 상태입니다. 수요 계획 → Overstock 필터 적용
긴급 발주 위험(Critical) 상태이면서 입고 예정 컨테이너가 없는 SKU 수. 즉시 발주가 필요합니다. 수요 계획 → Critical + 입고 없음 필터
지연 컨테이너 ETA가 이미 지났는데 아직 입고 완료되지 않은 컨테이너 수. 전체 카테고리 기준입니다. 컨테이너 일정 화면

② 3패널 현황

KPI 카드 아래에는 세 개의 패널이 나란히 표시됩니다. 각 패널의 "전체 보기 →" 링크를 클릭하면 상세 화면으로 이동합니다.

▲ 3패널 — 좌: 위험 SKU TOP 10 / 중: 입고 예정 컨테이너 / 우: 재고 분포

🔴 위험 SKU TOP 10

선택한 카테고리에서 SOD가 가장 짧은 SKU를 최대 10개 보여줍니다. SKU명을 클릭하면 SKU 계획 화면으로 바로 이동합니다. 각 행에 일평균 판매량(Daily Avg)이 함께 표시됩니다.

🚢 입고 예정 컨테이너

ETA 순으로 정렬된 예정 컨테이너 목록입니다. 컨테이너명 클릭 시 컨테이너 일정으로 이동합니다. ETA가 이미 지난 항목은 빨간색으로 표시됩니다. 상태 배지(Draft / Packing / Shipped / Complete)로 진행 단계를 한눈에 확인할 수 있습니다.

🍩 재고 분포 (Stock Health)

선택 카테고리의 SKU를 SOD 구간별로 나누어 도넛 차트로 시각화합니다.

  • ■ 위험 — SOD ≤ 30일
  • ■ 품절 예상 — 31~60일
  • ■ 정상 — 61~180일
  • ■ 과잉 재고 — 181일+

③ 판매 추세 차트

전체 채널의 판매 추세를 차트로 표시합니다. 차트 영역 전체를 클릭하면 주문 내역 화면으로 이동합니다.

▲ 판매 추세 — 기간·뷰·차트 유형 선택 가능
컨트롤설명
기간 선택 최근 7일 / 30일 / 90일 / 6개월 / 1년 / 올해 누계(YTD) 중 선택합니다. 선택한 기간과 직전 동일 기간을 자동으로 비교합니다.
매출 / 주문 토글 Y축을 매출(Revenue, $) 또는 주문 수량(Orders)으로 전환합니다.
차트 유형 영역(Area) · 선(Line) · 막대(Bar) 중 선택합니다. 원하는 시각화 스타일로 자유롭게 변경할 수 있습니다.

차트 오른쪽의 판매 요약 카드에는 선택 기간의 총 주문 수, 총 매출, 전기 대비 성장률(%)이 표시됩니다.

④ 새로고침

우측 상단의 ↺ 새로고침 버튼을 클릭하면 KPI 카드, 위험 SKU, 컨테이너 목록을 서버에서 최신 데이터로 다시 불러옵니다. 마지막 동기화 시각은 페이지 제목 아래에 표시됩니다.

전체 활성 SKU를 세그먼트별로 분류하고 예측 모델 성과를 한눈에 파악하는 페이지입니다.

② 세그멘테이션 개요

페이지 상단에는 전체 SKU가 세 가지 세그먼트로 나뉘어 요약됩니다. 예측 파이프라인이 실행될 때마다 각 SKU의 판매 이력을 분석해 세그먼트가 자동으로 재결정됩니다.

SKU 분류 기준

시스템이 각 SKU의 판매 이력을 보고 두 가지를 확인합니다.

  • 판매 없는 주의 비율: 전체 기간 중 아예 한 개도 팔리지 않은 주가 얼마나 되는지
  • 주당 평균 판매량: 한 주에 평균 몇 개나 팔리는지

⑤ Complete 상태와 Demand Planning 표시 규칙

⚠️
Complete 상태 컨테이너는 Demand Planning에 표시되지 않습니다.
컨테이너를 Complete로 변경하는 순간, 해당 컨테이너는 Demand Planning의 컨테이너 열 목록에서 즉시 사라지고, SKU별 발주 계획(Con. Qty) 계산에서도 제외됩니다.

시스템이 이렇게 동작하는 이유는 입고가 완료된 재고는 이미 창고 실재고(Current Stock)에 반영되었기 때문입니다. 이후 발주 계획은 갱신된 실재고 수치를 기준으로 다시 계산하면 됩니다.

🚫 Demand Planning에서 제외되는 조건

  • 상태 = Complete
  • 컨테이너 열이 사라지고 Con. Qty 계산에서 제외
  • 자동계산 버튼(📅 / 📊) 결과에도 미반영
  • SKU Planning > 재고 및 입고 탭의 입고 컨테이너 목록에도 미표시

✅ Demand Planning에 표시되는 조건

  • 상태 = Final List Sent
  • 상태 = Shipped (Packing List Rcvd)
  • 상태 = Draft (Draft 포함 토글 ON 시에만)

Draft → Final List Sent → Shipped (Packing List Rcvd) → Complete ❌

상황권장 조치
실제 입고가 완료된 컨테이너 Complete로 변경 → 창고 재고에 반영되었으므로 Demand Planning에서 자동 제외. 이후 재고 현황은 Demand Planning의 현재 재고(Stock) 수치로 확인.
아직 창고 입고 전인데 Complete로 잘못 변경한 경우 Container Planning에서 해당 컨테이너를 찾아 상태를 Shipped (Packing List Rcvd)로 되돌리면 Demand Planning에 즉시 다시 표시됩니다.
Complete 컨테이너 내역을 다시 보고 싶은 경우 Container Planning 페이지에서 직접 조회합니다. Demand Planning과 SKU Planning에서는 Complete 컨테이너가 표시되지 않습니다.
✅
올바른 Complete 처리 순서:
  1. 실물 입고 확인 후 Container Planning에서 해당 컨테이너 클릭
  2. 상태를 Complete로 변경
  3. 외부 재고 DB 동기화 후 Demand Planning에서 🔄 Sync 클릭
  4. 현재 재고(Stock) 수치가 갱신된 것을 확인하고 발주 계획을 재검토
⚠️
CBM 초과 주의: 화면 상단에 현재 CBM 사용률이 표시됩니다. 80 CBM을 초과하지 않도록 주의하세요.

A. 다중 컨테이너 통합 Export (Excel Export)

복수의 컨테이너를 동시에 선택해 하나의 엑셀 파일로 내보냅니다. 컨테이너별 발주 수량을 한눈에 비교하거나 공장에 Final Order를 전달할 때 사용합니다.

1

컨테이너 선택

컨테이너 목록 좌측의 체크박스를 클릭해 Export할 컨테이너를 선택합니다. 여러 개 동시 선택 가능합니다.

목록 하단의 Select visible 체크박스를 사용하면 현재 필터 조건에 맞는 컨테이너 전체를 한 번에 선택·해제할 수 있습니다. 선택된 개수는 옆에 N selected로 표시됩니다.

2

Excel Export 버튼 클릭

상단 헤더 우측의 Excel Export (N) 버튼(파란색 테두리)을 클릭합니다. N은 현재 선택된 컨테이너 수입니다. 0개 선택 시 버튼이 비활성화됩니다.

3

파일 다운로드

엑셀 파일이 자동으로 다운로드됩니다.

파일명 규칙

{제품명} ETA {M.DD.YYYY} ({첫 컨테이너} - {마지막 컨테이너} FINAL).xlsx
예시: Seat Cover ETA 8.15.2026 (SC-001 - SC-003 FINAL).xlsx
파일명 요소결정 방식
제품명선택된 컨테이너가 단일 제품군(Seat Cover, Car Cover 등)이면 해당 제품명 사용. 복수 제품군이 섞이면 "Mixed Product"로 표시.
ETA선택된 컨테이너 중 가장 이른 ETA 날짜를 M.DD.YYYY 형식으로 표기.
컨테이너 범위컨테이너 번호 기준 정렬 후 첫 번째~마지막 번호. 단일 선택 시 동일하게 표시.

B. 단일 컨테이너 Packing List Export (CSV/Excel)

개별 컨테이너의 SKU 목록을 패킹 리스트 형식으로 내보냅니다. 공장에 패킹 상세를 전달하거나 내부 검토용으로 사용합니다.

1

컨테이너 상세 열기

목록에서 컨테이너를 클릭해 우측 상세 패널을 엽니다.

2

CSV/Excel 버튼 클릭

상세 패널 하단 버튼 영역의 CSV/Excel 버튼을 클릭합니다.

이 버튼은 Draft 및 Final List Sent 상태에서만 표시됩니다. Shipped (Packing List Rcvd) 이상(입고 확정 후)에는 버튼이 숨겨집니다.

파일명 규칙

container-{컨테이너번호}-{오늘날짜}.xlsx
예시: container-SC-001-2026-06-15.xlsx

컨테이너 일정

입고 예정 컨테이너를 표시 기준일(입고 확정일 → ETA LAX/LGB → ETA 순으로 우선 적용)로 시각화합니다. Gantt 뷰(3M / 6M / 전체), 월별 달력 보기, 스케줄 보기 3가지 뷰를 드롭다운으로 전환할 수 있으며, 컨테이너를 클릭하면 SKU 상세 정보를 바로 확인할 수 있습니다.

▲ Container Timeline – 표시 기준일 기준 Gantt 뷰. 상태·상품 필터 · 기간 버튼 · 검색으로 표시 범위를 제어하며, 월별 헤더에 해당 월의 컨테이너 수가 표시됩니다.
📌
표시 기준일이란? 모든 뷰(Gantt · 월별 달력 · 스케줄)는 컨테이너마다 다음 우선순위로 하나의 날짜를 골라 기준으로 사용합니다: ① 입고 확정일(Confirmed Delivery, 있는 경우 항상 우선) → ② ETA LAX/LGB(항구 도착 예정일) → ③ ETA(창고 입고 예정일). 입고가 확정된 컨테이너는 칩·행에 ✓ 표시와 함께 확정일이 표시되고, 확정 전 컨테이너는 ETA LAX/LGB(없으면 ETA)가 표시됩니다.

① 화면 구성

🔍 필터 바

화면 상단에 검색창, 상태 필터 pills(다중 선택), 상품 필터 pills, 기간 선택 버튼이 위치합니다. 조건을 변경하면 Gantt 표가 즉시 업데이트됩니다.

📊 Gantt 표

월별로 그룹화되어 표시됩니다. 월 헤더 옆에 (N) 형식으로 해당 월의 컨테이너 수가 표시됩니다. 각 행은 컨테이너 하나를 나타내며 상태별로 색상이 구분됩니다.

📋 상세 드로어

Gantt 바 또는 행을 클릭하면 우측에 상세 패널이 슬라이드로 열립니다. SKU 목록, CBM 적재율, 메타 정보를 확인할 수 있습니다.

📋 목록 보기 이동

상단의 ← 목록 버튼을 클릭하면 현재 검색어·필터 상태 그대로 컨테이너 계획(목록) 페이지로 이동합니다.

③ Gantt 바 읽기

항목설명
바(Bar) 색상 Draft 빨강  |  Final List Sent 주황(amber)  |  Shipped (Packing List Rcvd) 파랑  |  Complete 녹색

바의 위치X축의 날짜 기준으로 표시 기준일(입고 확정일 → ETA LAX/LGB → ETA 순으로 적용)에 오른쪽 끝이 맞춰집니다.

④ 월별 달력 보기

보기 드롭다운에서 월별 보기를 선택하면 Google 캘린더 스타일의 한 달 달력으로 전환됩니다. 각 날짜 칸에 표시 기준일(입고 확정일 → ETA LAX/LGB → ETA 순으로 적용)이 해당하는 컨테이너 칩이 최대 4개까지 표시되며, 초과분은 +N개 더보기 버튼으로 확인할 수 있습니다. 컨테이너 타임라인 수정 권한이 있는 사용자는 칩을 다른 날짜 칸으로 드래그 앤 드랍해 날짜를 바로 변경할 수 있습니다.

▲ 월별 달력 보기 — 날짜 칸에 표시 기준일 컨테이너 칩이 표시되고, 헤더 오른쪽에 해당 월 전체 컨테이너 수가 표시됩니다.

드래그 앤 드랍으로 날짜 변경
(권한 필요)		컨테이너 타임라인 수정 권한(Role Permissions › Planning › 컨테이너 타임라인 › 수정)이 있는 사용자만 칩을 드래그할 수 있습니다. 칩을 원하는 날짜 칸에 드롭하면 해당 컨테이너의 날짜가 즉시 변경됩니다 — 입고가 확정된 컨테이너는 입고 확정일이 바뀌고, 그 외 컨테이너는 ETA LAX/LGB 날짜가 바뀝니다. 변경은 화면에 즉시 반영되며, 저장에 실패하면 원래 날짜로 되돌아가고 오류 메시지가 표시됩니다. 권한이 없는 사용자는 칩을 드래그할 수 없고 클릭으로 상세만 조회할 수 있습니다.

⑤ 스케줄 보기

보기 드롭다운에서 스케줄 보기를 선택하면 월 단위의 날짜별 그룹 리스트 뷰로 전환됩니다. 표시 기준일(입고 확정일 → ETA LAX/LGB → ETA 순으로 적용)을 기준으로 컨테이너가 날짜별로 묶여 시간순으로 나열되며, 각 컨테이너 행을 클릭하면 상세 드로어가 열립니다. (드래그 앤 드랍으로 날짜를 변경하는 기능은 월별 보기에서만 지원됩니다.)

▲ 스케줄 보기 — 표시 기준일별로 그룹화된 컨테이너 목록. 헤더 파란 배지에 해당 월 전체 컨테이너 수가 표시됩니다.
항목설명
월 헤더월 이름 옆 파란 배지에 표시 중인 월의 전체 컨테이너 수, ‹ / › 버튼으로 월 이동, 오늘 버튼으로 현재 월로 즉시 이동합니다.
날짜 그룹표시 기준일을 기준으로 컨테이너가 날짜별로 묶여 시간순으로 표시됩니다. 날짜 그룹 헤더는 요일 정보도 함께 표시됩니다.
컨테이너 행컨테이너 번호, 상태 배지, 공장명, SKU 수, CBM 적재율이 한 줄로 표시됩니다. 행을 클릭하면 상세 드로어가 열립니다.
빈 달 안내해당 월에 표시할 컨테이너가 없으면 안내 메시지와 함께 월 변경 또는 필터 조정을 권장합니다.

재고 이동 (Transit Stock)

창고 간 이동 중인 SKU와 수량을 기록·추적하고, 수요 계획 그리드에 반영합니다.

재고 이동 페이지는 한 창고에서 다른 창고로 이동 중인 재고를 관리하는 운영 로그입니다. 등록된 이동 중(In Transit) 레코드의 수량 합계가 수요 계획 그리드의 Transit Stock 컬럼에 자동 반영됩니다.

ℹ️
Transit Stock과 수요 계획: 레코드를 추가·삭제·도착 완료 처리하면 즉시 fc_stats.transit_stock이 재계산되어 수요 계획 그리드에 반영됩니다. 별도 Refresh 불필요.

Basic Workflow

1

Verify SKU Master

CBM (volume), MOQ, and order multiples must be correctly registered for purchase calculations to work. Always check this first when adding a new SKU.

2

Register Containers in Container Planning

Add inbound containers and enter the ETA (estimated arrival date). Manage SKU quantities as a packing list.

3

Plan Orders in Demand Planning

Review inventory status and container chains, then use auto-fill buttons to calculate optimal order quantities.

4

Allocate Surplus in Available Stock

5

Analyze Details in SKU Planning

Select a SKU to review its sales trends and estimated sell-out date.

Forecast Model Guide

Use Auto unless you have a specific reason to choose otherwise.

🤖 Auto (Recommended)

Tests all models against past sales and picks the one with the smallest error. Use this unless you have a specific reason not to.

📊 AutoARIMA

Good for products that follow a seasonal pattern — selling more at certain times of year.

📈 AutoETS

Good for products whose sales are gradually trending up or down.

📉 AutoTheta

Good for products that are steadily growing or declining over a long period.

📋 Window Avg

Uses the average of the most recent N weeks. Works well when sales are steady with no clear trend.

📋 Historic Avg

Uses the average of all past sales. Best when sales have stayed roughly the same for a long time.

🔄 Seasonal Naïve

Copies last year's sales for the same week. Best when the same seasonal pattern repeats every year.

➡️ Naïve

Repeats last week's number. Used for very erratic products or as a comparison baseline.

⑤ Complete Status and Demand Planning Display Rules

⚠️
Complete containers are not shown in Demand Planning.
The moment a container is changed to Complete, it immediately disappears from the Demand Planning container column list and is excluded from per-SKU Con. Qty calculations.

This behavior exists because inventory from a received container is already reflected in the warehouse's live stock figures (Current Stock). Subsequent order planning should be recalculated against the updated live stock numbers.

🚫 Conditions that Exclude from Demand Planning

  • Status = Complete
  • Container column disappears; Con. Qty excluded from calculations
  • Not reflected in Auto-Fill results (📅 / 📊)
  • Not shown in SKU Planning › Inventory & Inbound container list

✅ Conditions that Show in Demand Planning

  • Status = Final List Sent
  • Status = Shipped (Packing List Rcvd)
  • Status = Draft (only when Include Draft toggle is ON)

Demand Planning Changes When Status Transitions

Draft → Final List Sent → Shipped (Packing List Rcvd) → Complete ❌
