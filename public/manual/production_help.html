전체 그림

4개 화면이 서로 어떻게 연결되는지 먼저 이해하면 헷갈리지 않습니다.

Vehicles(차종)와 Parts & Codes(부품/코드/이니셜 사전)는 각각 독립된 마스터 데이터입니다. Part SKU Generator가 이 둘을 조합해서 하나의 SKU 문자열을 만듭니다.

🚗 Vehicles

Make / Model 등 차종 정보. Part SKU Generator의 차종 선택 목록으로 쓰입니다.

🔤 Parts & Codes

부품명 / 코드 / 디자이너 이니셜 사전. SKU 조합의 재료입니다.

🏷️ Part SKU Generator

위 둘을 조합해 SKU를 만들고 저장·관리합니다.

🧵 시트 커버 부품

완전히 별도 기능입니다. 사이즈별 시트 커버 부품 구성(BOM)만 관리하며, 나머지 3개와 데이터로 연결되지 않습니다.

💡
SKU를 새로 만들려면 순서가 있습니다: ① Vehicles에 차종 등록 → ② Parts & Codes에 부품/코드/이니셜 등록 → ③ Part SKU Generator에서 조합. 순서를 지키면 드롭다운에 원하는 값이 바로 나타납니다.

시트 커버 부품

시트 커버 사이즈별 부품 구성(BOM)을 Front/Rear/Third 좌석 위치별로 관리합니다.

▲ Front / Rear / Third 탭 + 부품 그리드

하나의 Size(시트 사이즈 코드)마다 Headrest · Top/Body · Bottom · Middle · Armrest 등 부품 정보를 Driver/Passenger 구분과 함께 등록합니다. Rear/Third 탭에는 Console · Backrest Storage · Subpart 항목이 추가로 있습니다.

사용법

1

Front / Rear / Third 탭 선택

좌석 위치별로 별도 그리드가 열립니다.

2

행 선택 후 추가 / 수정 / 삭제

Driver/Passenger 드롭다운("동일" / "Driver 기준 대칭" / "Passenger 기준 대칭")을 고르면 좌우 필드가 자동으로 채워집니다.

3

저장

입력한 부품 값들을 조합한 Package 요약 텍스트가 자동으로 함께 저장됩니다.

각 컬럼 헤더의 검색창으로 필터링할 수 있고, 현재 탭에 보이는 데이터만 엑셀로 내보낼 수 있습니다.

⚠️
이 화면은 Vehicles·Parts & Codes·Part SKU Generator와 데이터로 연결되어 있지 않은 완전히 별도의 마스터입니다. Size 값은 여기서만 의미를 가집니다.

Vehicles

차종(Make/Model) 마스터를 조회·등록·수정하고, 외부 시스템과 동기화합니다.

▲ F Number · Make · Model · Submodel 1~6 · Add/Edit/Sync
컬럼의미
F Number차량 고유 식별자 (등록 후 수정 불가)
Year/Generation연식·세대 정보
Make / Model / Model 2제조사 · 모델명
Submodel 1~6트림 등 세부 구분 (라벨+값 쌍, 최대 6개)

Sync (동기화)

Sync 버튼을 누르면 외부 차량 데이터 소스에서 최신 목록을 가져와 전체 upsert + 소스에서 사라진 항목은 삭제하는 방식으로 동기화합니다. 동기화 후 "upserted: n, deleted: m" 결과가 표시됩니다.

⚠️
개별 행 삭제 버튼은 없습니다. 특정 차량을 지우려면 원천 데이터에서 제거한 뒤 Sync를 실행해야 합니다.

여기 등록된 Make/Model은 Part SKU Generator의 차종 드롭다운에 그대로 나타납니다.

Parts & Codes

Part(부품명) · Code(코드) · Designer Initial(디자이너 이니셜) 3종의 사전 데이터를 관리합니다.

▲ Part / Code / Designer Initial 탭 — 검색 + 목록/상세

이 화면 자체는 업무 트랜잭션을 만들지 않습니다. Part SKU Generator가 SKU를 조합할 때 참조하는 "허용 값 사전"입니다.

탭내용
Part부품명 + 설명. 부품명 자체가 고유 식별자입니다.
Code코드 + 설명. 저장 시 자동으로 대문자로 정규화됩니다.
Designer Initial이니셜 + 담당 디자이너 이름. 저장 시 자동으로 대문자로 정규화됩니다.

사용법

1

탭 선택 (Part / Code / Designer Initial)

세 탭 모두 같은 방식(검색 → 목록 → 상세 → 추가/수정/삭제)으로 동작합니다.

2

추가·수정 시 활성(active) 여부 설정

비활성 처리된 항목은 Part SKU Generator의 선택 목록에 더 이상 나타나지 않습니다.

3

"비활성 포함" 체크박스로 과거 항목도 조회

삭제는 실제 삭제가 아니라 비활성 처리이므로 언제든 다시 활성화할 수 있습니다.

상태의미
활성Part SKU Generator 드롭다운에 노출됨
비활성드롭다운에서 숨겨지지만 데이터는 보존됨

Part SKU Generator

차종·부품·코드·이니셜·좌우 구분을 조합해 표준 SKU를 생성하고 관리합니다.

▲ "생성기" 탭 — 1열(Front)/2열(Rear)/3열(Third Row) 좌석 다이어그램에서 부위를 클릭

화면 상단에 생성기 / 목록 탭이 있습니다. 기본으로 열리는 "생성기" 탭은 좌석 다이어그램을 클릭해 부품을 찾는 방식으로 동작합니다.

SKU 조합 규칙

{부품명}-{Make 약어}-{Model 약어}-{코드}-{이니셜}[-{좌우구분}]

좌우 구분(Side)이 Universal이면 SKU 마지막 구분 세그먼트는 생략됩니다.

사용법 — 다이어그램에서 부위를 클릭해 Part를 찾고, 세션으로 SKU를 완성

1

1열/2열/3열 토글 후 다이어그램에서 부위 클릭

각 좌석은 헤드레스트 · 등받이 · 방석 · 팔걸이로 구성되고, 2열/3열에는 등받이 수납·서브파트·콘솔이 추가됩니다. 부위를 클릭하면(다시 클릭하면 선택 해제) 그 부위 조건에 맞는 Part 목록이 아래에 나타납니다.

2

목록에서 실제 Part 클릭

부위는 검색 필터일 뿐이고, 이 단계에서 고른 Part가 SKU의 부품명이 됩니다.

3

우측 패널에서 세션 설정: Make → Model → Make Abbr → Model Abbr → Initial

Make를 고르면 Model 목록이 그 Make 기준으로 다시 채워집니다. Make Abbr / Model Abbr는 자유 입력 텍스트(예: "HY", "PA")입니다.

4

Code / Side 선택 → SKU 미리보기 확인 → "+ 생성"

세션이 준비되면 Code·Side가 활성화되고, 값이 모두 채워지면 SKU 미리보기가 실시간으로 나타납니다. 저장하면 Part/Code/Side만 초기화되고 Make/Model/Initial 세션은 유지된 채 "목록" 탭으로 자동 전환되어 방금 만든 SKU가 선택됩니다.

⚠️
Make Abbr / Model Abbr는 사전 정의된 목록이 아니라 자유 입력입니다. 같은 차종에 다른 약어(예: "HY" vs "Hyundai")를 입력하면 SKU 표기가 일관되지 않으니, 팀 내에서 약어 표기를 미리 통일해두는 것이 좋습니다.
🧩
다이어그램의 부위 이름(헤드레스트·등받이·방석·팔걸이·콘솔·등받이 수납·서브파트)은 시트 커버 부품 화면과 같은 분류 체계를 재사용한 것일 뿐, 실제 데이터는 서로 다른 곳(이 화면은 Parts & Codes의 Part 데이터, 시트 커버 부품은 별도 사이즈별 마스터)에 저장됩니다. 이름이 비슷하다고 같은 데이터는 아닙니다.

목록 탭 — 생성 후 관리

  • 좌측 목록에서 SKU/부품명으로 검색, "비활성 포함" 체크로 지난 항목도 조회
  • 우측 상세에는 구성 요소(좌석 위치/Part/Make/Model/Code/Initial/Side/생성자) breakdown과 체크리스트(진행 항목 추가·Pending/In Progress/Done 상태 변경·삭제)
  • 삭제 버튼은 실제 삭제가 아니라 비활성 처리(soft delete) — 완전히 지워지지 않고 "비활성 포함" 조회로 다시 볼 수 있습니다

참고

자주 겪는 문제

헷갈리기 쉬운 동작을 미리 정리했습니다.

증상원인 / 해결
Part SKU Generator 드롭다운에 원하는 차종/부품이 안 보임 Vehicles 또는 Parts & Codes에 해당 값이 등록·활성화되어 있는지 확인하세요. 비활성(off) 상태면 목록에 나타나지 않습니다.
같은 SKU를 다시 만들었는데 중복으로 안 걸림 중복 체크는 SKU 문자열이 완전히 같을 때만 걸립니다. Make Abbr 표기가 다르면(예: "HY" vs "Hyundai") 다른 SKU로 인식됩니다 — 약어 표기를 통일해서 사용하세요.
Vehicles에서 특정 차량만 지우고 싶음 개별 삭제 기능이 없습니다. 원천 데이터에서 제거한 뒤 Sync를 실행하면 함께 정리됩니다.
비활성 처리한 Part/Code/SKU를 되돌리고 싶음 모두 소프트 삭제라 데이터가 남아있습니다. "비활성 포함" 체크박스로 찾아서 다시 활성화하세요.
